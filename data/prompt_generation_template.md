# Шаблон промта для генерации статей нового типа

## Описание задачи
Вы являетесь экспертом в области программирования и технической документации. Ваша задача - создать подробную статью/документацию по заданной теме, оформленную в формате текстового файла с HTML-разметкой. Каждый файл должен содержать исчерпывающую информацию о конкретной теме, включая теорию, примеры кода и практические рекомендации.

## Типы статей
Сайт поддерживает следующие категории статей:
- Паттерны проектирования (design patterns)
- Архитектурные паттерны
- Элементы STL
- Микроконтроллеры и встраиваемые системы
- Алгоритмы
- Языковые особенности C++
- Инструменты и технологии разработки
- Стандарты C++

## Файловая структура и организация данных
Все статьи организованы в древовидную структуру с помощью файлов index.json:
- Каждая категория статей имеет свою папку в корне папки data
- В каждой папке находится index.json, который описывает элементы этой категории
- Некоторые элементы могут иметь подкатегории, для которых создаются дополнительные папки
- Конечные статьи представлены в виде .txt файлов

### Структура файлов index.json
Каждый файл index.json содержит массив объектов со следующими полями:
- type: тип категории (например, "stl", "patterns", "architectures")
- name: отображаемое название элемента
- fileName: имя файла (без расширения), в котором находится содержимое статьи
- category: категория элемента
- usage: краткое описание назначения элемента
- isLeaf: флаг, указывающий, является ли элемент конечным (true) или содержит подэлементы (false)

Если isLeaf равно false, то для элемента создается подпапка с именем из поля fileName, в которой также будет находиться index.json с подэлементами.

## Формат файла
- Файл должен быть в формате .txt
- имя файла должно быть в формате `index.txt`
- блоки кода должны иметь имя в формате `<тема>_<язык>_<номер(Если на несколько поделен)>.txt`
- Может содержать HTML-теги для форматирования (но не для кода)
- Должен включать в себя теоретическую часть, примеры кода и объясненияф
- Размер статьи: от 500 до 2000 слов

## Структура статьи
1. Заголовок (h2 или h3)
2. Введение в тему (1–2 абзаца)
3. Подробное объяснение темы:
4. Теоретическая база

5. Примеры кода с пояснениями
Важно: все примеры кода должны размещаться в специальном формате:"
<pre><code>
имяфайла.txt
<file>
... код ...
</file>
</code></pre>"

Практические применения

Особенности и подводные камни

Заключение (краткое резюме)

Ссылки (по необходимости)
## Требования к содержанию
- Использовать профессиональную, но понятную терминологию
- Включать актуальные примеры кода на C++ (для соответствующих тем)
- Учитывать различные версии стандарта C++ (C++11, C++14, C++17, C++20, C++23)
- Отмечать, если какие-то функции относятся к конкретной версии стандарта
- Придерживаться лучших практик программирования
- Указывать потенциальные проблемы и способы их избежать

## Требования к оформлению
- Использовать HTML-теги для структурирования контента (заголовки, параграфы, списки)
- Примеры кода вставлять с помощью специального блока с именем файла, который будет загружаться по требованию
- Важные термины можно выделять тегом <strong> (опционально)
- Использовать списки (ul/ol) для перечислений
- При необходимости использовать таблицы для сравнения

## Дополнительные инструкции
- Статья должна быть самодостаточной и не требовать дополнительных источников
- Если тема позволяет, привести сравнение с аналогами
- Указать лучшие практики использования
- Обозначить ограничения и потенциальные проблемы
- Рекомендовать дополнительные источники информации по теме
- Для каждого примера кода создавать отдельный txt-файл
- В основном файле использовать блок в формате показанном выше для вставки кода
- Таких блоков с кодом может быть несколько в одном файле
