<h2>HAL / LL / CMSIS</h2>

<p><strong>Категория:</strong> Программирование</p>

<h3>Описание</h3>
<p>HAL (Hardware Abstraction Layer), LL (Low Layer) и CMSIS (Cortex Microcontroller Software Interface Standard) - это уровни программной абстракции, предоставляемые STMicroelectronics и ARM для разработки приложений на микроконтроллерах STM32. Эти библиотеки обеспечивают различные уровни абстракции и гибкости при программировании периферии микроконтроллера.</p>

<h3>HAL vs LL vs CMSIS:</h3>
<p>Три уровня программного обеспечения предоставляют разные преимущества в зависимости от требований к производительности, времени выполнения и удобству разработки:</p>

<ul>
<li><strong>CMSIS</strong> - самый низкий уровень, предоставляет прямой доступ к регистрам ядра ARM Cortex-M. Предоставляет функции для работы с системными аспектами ядра (NVIC, SysTick, MPU и т.д.).</li>
<li><strong>LL</strong> - библиотека с высокоэффективными макросами и встроенными функциями, которые обеспечивают прямой доступ к регистрам периферии. Обеспечивает оптимизированный по производительности код с небольшим размером.</li>
<li><strong>HAL</strong> - самый высокий уровень абстракции, обеспечивающий переносимость кода между различными семействами STM32. Обеспечивает удобство разработки за счет дополнительных накладных расходов.</li>
</ul>

<h3>Конфигурация периферии:</h3>
<p>Конфигурация периферии может выполняться через каждый из уровней, каждый со своими особенностями:</p>

<h4>Пример конфигурации UART через HAL:</h4>

<pre>
<code>
#include "stm32f4xx_hal.h"

UART_HandleTypeDef huart2;

void UART2_Init_HAL(void) {
    // Включить тактирование UART2 и GPIOA
    __HAL_RCC_USART2_CLK_ENABLE();
    __HAL_RCC_GPIOA_CLK_ENABLE();

    // Настроить структуру GPIO
    GPIO_InitTypeDef gpio_init = {0};
    gpio_init.Pin = GPIO_PIN_2 | GPIO_PIN_3;  // TX:PA2, RX:PA3
    gpio_init.Mode = GPIO_MODE_AF_PP;
    gpio_init.Pull = GPIO_NOPULL;
    gpio_init.Speed = GPIO_SPEED_FREQ_VERY_HIGH;
    gpio_init.Alternate = GPIO_AF7_USART2;
    HAL_GPIO_Init(GPIOA, &gpio_init);

    // Настроить UART
    huart2.Instance = USART2;
    huart2.Init.BaudRate = 115200;
    huart2.Init.WordLength = UART_WORDLENGTH_8B;
    huart2.Init.StopBits = UART_STOPBITS_1;
    huart2.Init.Parity = UART_PARITY_NONE;
    huart2.Init.Mode = UART_MODE_TX_RX;
    huart2.Init.HwFlowCtl = UART_HWCONTROL_NONE;
    huart2.Init.OverSampling = UART_OVERSAMPLING_16;

    if (HAL_UART_Init(&huart2) != HAL_OK) {
        Error_Handler();
    }
}
</code>
</pre>

<h4>Пример конфигурации UART через LL:</h4>

<pre>
<code>
#include "stm32f4xx_ll_usart.h"
#include "stm32f4xx_ll_gpio.h"
#include "stm32f4xx_ll_bus.h"
#include "stm32f4xx_ll_rcc.h"

void UART2_Init_LL(void) {
    // Включить тактирование UART2 и GPIOA
    LL_APB1_GRP1_EnableClock(LL_APB1_GRP1_PERIPH_USART2);
    LL_AHB1_GRP1_EnableClock(LL_AHB1_GRP1_PERIPH_GPIOA);

    // Настроить GPIO
    LL_GPIO_SetPinMode(GPIOA, LL_GPIO_PIN_2, LL_GPIO_MODE_ALTERNATE);
    LL_GPIO_SetPinMode(GPIOA, LL_GPIO_PIN_3, LL_GPIO_MODE_ALTERNATE);
    LL_GPIO_SetPinOutputType(GPIOA, LL_GPIO_PIN_2, LL_GPIO_OUTPUT_PUSHPULL);
    LL_GPIO_SetPinOutputType(GPIOA, LL_GPIO_PIN_3, LL_GPIO_OUTPUT_PUSHPULL);
    LL_GPIO_SetPinSpeed(GPIOA, LL_GPIO_PIN_2, LL_GPIO_SPEED_FREQ_VERY_HIGH);
    LL_GPIO_SetPinSpeed(GPIOA, LL_GPIO_PIN_3, LL_GPIO_SPEED_FREQ_VERY_HIGH);
    LL_GPIO_SetPinPull(GPIOA, LL_GPIO_PIN_2, LL_GPIO_PULL_NO);
    LL_GPIO_SetPinPull(GPIOA, LL_GPIO_PIN_3, LL_GPIO_PULL_NO);
    LL_GPIO_SetAFPin_0_7(GPIOA, LL_GPIO_PIN_2, LL_GPIO_AF_7);
    LL_GPIO_SetAFPin_0_7(GPIOA, LL_GPIO_PIN_3, LL_GPIO_AF_7);

    // Настроить UART
    LL_USART_SetBaudRate(USART2, 
                         LL_RCC_GetPCLK1Freq(), 
                         LL_USART_OVERSAMPLING_16, 
                         115200);
    LL_USART_SetDataWidth(USART2, LL_USART_DATAWIDTH_8B);
    LL_USART_SetStopBitsLength(USART2, LL_USART_STOPBITS_1);
    LL_USART_SetParity(USART2, LL_USART_PARITY_NONE);
    LL_USART_SetTransferDirection(USART2, LL_USART_DIRECTION_TX_RX);
    LL_USART_SetHWFlowCtrl(USART2, LL_USART_HWCONTROL_NONE);

    // Включить UART
    LL_USART_Enable(USART2);

    // Ожидать окончания включения
    while (!LL_USART_IsActiveFlag_TEACK(USART2));
    while (!LL_USART_IsActiveFlag_REACK(USART2));
}
</code>
</pre>

<h4>Пример конфигурации UART через прямую работу с регистрами (CMSIS/ручной подход):</h4>

<pre>
<code>
#include "stm32f4xx.h"

void UART2_Init_Registers(void) {
    // Включить тактирование UART2 и GPIOA
    RCC->AHB1ENR |= RCC_AHB1ENR_GPIOAEN;
    RCC->APB1ENR |= RCC_APB1ENR_USART2EN;

    // Подождать включения тактирования
    while(!(RCC->AHB1ENR & RCC_AHB1ENR_GPIOAEN));
    while(!(RCC->APB1ENR & RCC_APB1ENR_USART2EN));

    // Настроить GPIO (PA2 и PA3)
    // Сначала сконфигурировать GPIOA pins 2 and 3 as alternate function
    GPIOA->MODER &= ~(GPIO_MODER_MODER2 | GPIO_MODER_MODER3); // Clear bits
    GPIOA->MODER |= (GPIO_MODER_MODER2_1 | GPIO_MODER_MODER3_1); // Set alternate function mode
    
    GPIOA->OTYPER &= ~(GPIO_OTYPER_OT_2 | GPIO_OTYPER_OT_3); // Push-pull
    
    GPIOA->OSPEEDR |= (GPIO_OSPEEDER_OSPEEDR2 | GPIO_OSPEEDER_OSPEEDR3); // High speed
    
    GPIOA->PUPDR &= ~(GPIO_PUPDR_PUPDR2 | GPIO_PUPDR_PUPDR3); // No pull-up, no pull-down
    
    GPIOA->AFR[0] |= (7 << (2 * 4)) | (7 << (3 * 4)); // Alternate function 7 for pins 2 and 3

    // Настроить UART
    // Установить скорость передачи данных (baud rate)
    // Установим OVER8=0 (oversampling by 16)
    USART2->CR1 &= ~USART_CR1_OVER8; // OVER8 = 0
    uint32_t pclk1 = 42000000; // предполагаем, что PCLK1 = 42MHz
    uint32_t usartdiv = (pclk1 * 25) / (4 * 115200); // формула для OVERSAMPLING_16
    uint32_t mantissa = usartdiv / 100;
    uint32_t fraction = ((usartdiv - (mantissa * 100)) * 16 + 50) / 100;
    USART2->BRR = (mantissa << 4) | (fraction & 0x0F);

    // Настроить параметры передачи: 8 бит, 1 стоп-бит
    USART2->CR1 &= ~(USART_CR1_M | USART_CR1_PCE); // 8 бит, без проверки четности
    USART2->CR2 &= ~USART_CR2_STOP; // 1 стоп-бит

    // Включить UART
    USART2->CR1 |= USART_CR1_UE; // Enable USART
    USART2->CR1 |= (USART_CR1_RE | USART_CR1_TE); // Enable receiver and transmitter
}
</code>
</pre>

<h3>CubeMX / CubeIDE:</h3>
<p>STM32CubeMX - это графический инструмент для конфигурирования микроконтроллера STM32. Он позволяет визуально настроить периферию, тактирование, GPIO и другие параметры, а затем генерирует инициализационный код.</p>

<ul>
<li>Визуальный дизайн проекта</li>
<li>Проверка конфликтов между периферийными модулями</li>
<li>Генерация инициализационного кода для HAL или LL</li>
<li>Управление проектом и библиотеками</li>
<li>Интеграция с IDE (CubeIDE, Keil, IAR и т.д.)</li>
</ul>

<h4>Преимущества CubeMX:</h4>
<ul>
<li>Ускоряет начальную настройку проекта</li>
<li>Помогает избежать ошибок конфигурации периферии</li>
<li>Автоматически генерирует код инициализации</li>
<li>Показывает назначение выводов и конфликты</li>
</ul>

<h3>Генерация кода:</h3>
<p>STM32CubeMX генерирует инициализационный код в нескольких файлах:</p>
<ul>
<li><strong>main.c</strong> - содержит основную функцию и инициализацию периферии</li>
<li><strong>stm32xxx_hal_conf.h</strong> - файл конфигурации HAL библиотеки</li>
<li><strong>stm32xxx_it.c</strong> - обработчики прерываний</li>
<li><strong>gpio.c/h</strong> - инициализация GPIO</li>
<li><strong>usart.c/h</strong> - инициализация UART (если используется)</li>
<li><strong>rcc.c/h</strong> - инициализация тактирования</li>
</ul>

<h3>Ручная настройка регистров:</h3>
<p>Ручная настройка регистров дает полный контроль над периферией и может привести к более эффективному коду, но требует более глубокого понимания работы микроконтроллера.</p>

<h4>Преимущества ручной настройки:</h4>
<ul>
<li>Меньший размер кода</li>
<li>Высокая производительность</li>
<li>Полный контроль над периферией</li>
<li>Лучшее понимание работы микроконтроллера</li>
</ul>

<h4>Недостатки ручной настройки:</h4>
<ul>
<li>Более высокий риск ошибок</li>
<li>Меньшая переносимость кода</li>
<li>Требуется больше времени на разработку</li>
<li>Необходимо часто обращаться к документации</li>
</ul>

<h3>Рекомендации по выбору подхода:</h3>
<ul>
<li>Для быстрого прототипирования и проектов с ограниченным временем разработки - использовать HAL</li>
<li>Для проектов с жесткими требованиями к производительности и размеру кода - использовать LL</li>
<li>Для критических участков кода или проектов с очень жесткими требованиями - использовать прямую работу с регистрами</li>
<li>Для обучения и глубокого понимания работы микроконтроллера - рекомендуется изучать все три подхода</li>
</ul>
