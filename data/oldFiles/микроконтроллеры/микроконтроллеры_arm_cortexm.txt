<h2>Архитектура ARM Cortex-M</h2>

<p><strong>Категория:</strong> Архитектуры</p>

<h3>Описание</h3>
<p>ARM Cortex-M - это серия 32-битных процессоров, разработанных компанией ARM Holdings для встраиваемых приложений. Эти процессоры отличаются высокой производительностью при низком энергопотреблении и простоте разработки программного обеспечения. Cortex-M процессоры находят широкое применение в современных микроконтроллерах таких как STM32, LPC, Kinetis и других.</p>

<h3>Основные особенности:</h3>
<ul>
<li>32-битная RISC архитектура с гарвардской структурой шин</li>
<li>Низкое энергопотребление и высокая производительность</li>
<li>Наличие NVIC (Nested Vectored Interrupt Controller) для эффективной обработки прерываний</li>
<li>Поддержка Thumb-2 технологии команд для оптимизации плотности кода</li>
<li>Встроенная система управления памятью (MPU) в некоторых версиях</li>
<li>Поддержка DSP инструкций в некоторых версиях (Cortex-M4/M7)</li>
<li>Встроенный FPU (Floating Point Unit) в версиях с DSP расширением</li>
</ul>

<h3>Семейства Cortex-M:</h3>
<ul>
<li>Cortex-M0/M0+ - самые простые и экономичные</li>
<li>Cortex-M3 - средний уровень, без FPU</li>
<li>Cortex-M4/M7 - с DSP расширениями, часто с FPU</li>
<li>Cortex-M33/M35P - с расширенными возможностями безопасности</li>
</ul>

<h3>Регистры Cortex-M:</h3>
<ul>
<li>R0-R12 - общего назначения</li>
<li>SP (Stack Pointer) - указатель стека</li>
<li>LR (Link Register) - регистр возврата из подпрограммы</li>
<li>PC (Program Counter) - счетчик команд</li>
<li>xPSR - регистр состояния процессора</li>
<li>CONTROL - регистр управления режимом процессора</li>
</ul>

<h3>Пример использования CMSIS для работы с системными параметрами:</h3>

<pre>
<code>
#include "stm32f4xx.h"  // Заголовочный файл для конкретного MCU
#include &lt;stdio.h&gt;

/* Пример использования CMSIS функций */
int main(void) {
    // Инициализация системной частоты
    SystemInit();
    
    // Получение текущей частоты шины APB1 (PCLK1)
    uint32_t pclk1_freq = HAL_RCC_GetPCLK1Freq();
    
    // Установка приоритета прерывания
    NVIC_SetPriority(USART2_IRQn, 0);
    
    // Включение прерывания
    NVIC_EnableIRQ(USART2_IRQn);
    
    // Получение текущего значения системного таймера
    uint32_t sys_tick_val = SysTick->VAL;
    
    // Установка значения перезагрузки системного таймера
    SysTick->LOAD = (pclk1_freq / 8 / 1000) - 1; // 1ms интервал
    
    // Включение системного таймера
    SysTick->CTRL = SysTick_CTRL_CLKSOURCE_Msk | 
                    SysTick_CTRL_TICKINT_Msk   | 
                    SysTick_CTRL_ENABLE_Msk;
    
    while(1) {
        // Основной цикл программы
    }
}

/* Обработчик прерывания системного таймера */
void SysTick_Handler(void) {
    // Обработка прерывания от системного таймера
    // Вызывается каждую миллисекунду
}
</code>
</pre>

<h3>Преимущества архитектуры Cortex-M:</h3>
<ul>
<li>Единая архитектура позволяет легче переносить код между разными производителями микроконтроллеров</li>
<li>Отладка реализована на уровне архитектуры (SWD интерфейс)</li>
<li>Богатая экосистема инструментов разработки и поддержка многих ОСРВ</li>
<li>Поддержка безопасности через MPU и TrustZone в новых версиях</li>
</ul>

<h3>ARM Cortex-A архитектура</h3>
<p>В дополнение к серии Cortex-M, ARM также предлагает серию Cortex-A процессоров, предназначенных для сложных вычислительных задач и работы с операционными системами, такими как Linux. Cortex-A процессоры реализуют архитектуру ARMv7-A или ARMv8-A (для 64-битных систем) и поддерживают режимы работы Normal (ранее назывался User) и Privileged (привилегированный).</p>
<ul>
<li>Поддерживают виртуальную память и страничную организацию памяти</li>
<li>Реализуют MMU (Memory Management Unit) для управления виртуальной памятью</li>
<li>Используются в приложениях с высокими требованиями к производительности: смартфоны, планшеты, встраиваемые системы с ОС Linux</li>
<li>Включают Advanced Microcontroller Bus Architecture (AMBA) с шинами AXI, AHB, APB</li>
</ul>

<h3>Режимы работы процессора</h3>
<p>Процессоры ARM поддерживают несколько режимов работы, которые определяют права доступа и доступные инструкции:</p>
<ul>
<li>User - обычный режим с ограниченными правами</li>
<li>System - режим для операционных систем, с теми же привилегиями, что и Supervisor</li>
<li>Supervisor - режим с высокими привилегиями для ОС</li>
<li>FIRQ и IRQ - режимы обработки прерываний</li>
<li>Abort - режим для обработки ошибок доступа к памяти</li>
<li>Undefined - режим для обработки неопределенных инструкций</li>
</ul>

<h3>Система шин (AHB/APB)</h3>
<p>ARM процессоры используют систему шин для связи различных компонентов:</p>
<ul>
<li>AHB (Advanced High-performance Bus) - высокопроизводительная шина для подключения быстрых периферийных устройств</li>
<li>APB (Advanced Peripheral Bus) - низкоскоростная шина для подключения менее требовательных периферийных устройств (UART, GPIO, таймеры)</li>
<li>AXI (Advanced eXtensible Interface) - используется в Cortex-A процессорах для высокопроизводительного взаимодействия</li>
</ul>

<h3>MPU (Memory Protection Unit)</h3>
<p>MPU - это компонент, обеспечивающий базовую защиту памяти в системах без MMU. Он позволяет определять области памяти с различными атрибутами доступа и управления кэшированием:</p>
<ul>
<li>Разграничение доступа к различным областям памяти</li>
<li>Защита от перезаписи критических участков памяти</li>
<li>Настройка атрибутов кэширования для разных регионов</li>
<li>Поддержка доверенных и недоверенных задач в RTOS</li>
</ul>

<h3>FPU (Floating Point Unit)</h3>
<p>FPU - это аппаратный сопроцессор для выполнения операций с плавающей точкой. ARM предлагает два стандарта FPU:</p>
<ul>
<li>Single Precision (32-битные числа с плавающей точкой)</li>
<li>Double Precision (64-битные числа с плавающей точкой)</li>
<li>Поддержка форматов IEEE 754</li>
<li>Ускорение математических вычислений, особенно тригонометрических функций и вычислений с датчиков</li>
</ul>

<h3>CMSIS (Cortex Microcontroller Software Interface Standard)</h3>
<p>CMSIS - это набор стандартов программного интерфейса для микроконтроллеров ARM Cortex-M. CMSIS включает в себя несколько компонентов:</p>
<ul>
<li>CMSIS-Core - предоставляет интерфейс к ядру Cortex-M и периферии</li>
<li>CMSIS-DSP - библиотека цифровой обработки сигналов</li>
<li>CMSIS-RTOS - API для работы с RTOS</li>
<li>CMSIS-Pack - формат описания компонентов и периферии</li>
<li>CMSIS-DAP - интерфейс для отладки</li>
</ul>

<h3>Пример использования CMSIS DSP библиотеки:</h3>

<pre>
<code>
#include "arm_math.h"
#include &lt;stdio.h&gt;

int main(void) {
    // Пример вычисления FFT с использованием CMSIS DSP
    #define SIGNAL_LENGTH 1024
    
    float32_t input[SIGNAL_LENGTH * 2];  // Входной массив (вещественные и мнимые части)
    float32_t output[SIGNAL_LENGTH];     // Выходной массив
    uint32_t fft_index;
    float32_t maxValue;
    
    // Инициализация функции FFT
    arm_cfft_radix4_instance_f32 fft_inst;
    arm_cfft_radix4_init_f32(&fft_inst, SIGNAL_LENGTH, 0, 1);
    
    // Заполнение входного массива (реальные данные будут отличаться)
    for(int i = 0; i < SIGNAL_LENGTH; i++) {
        input[2*i] = /* вещественная часть */;      // Заполнить реальными данными
        input[2*i+1] = /* мнимая часть */;          // Заполнить реальными данными
    }
    
    // Выполнение FFT
    arm_cfft_radix4_f32(&fft_inst, input);
    
    // Вычисление мощности сигнала
    arm_cmplx_mag_f32(input, output, SIGNAL_LENGTH);
    
    // Нахождение максимального значения
    arm_max_f32(output, SIGNAL_LENGTH, &maxValue, &fft_index);
    
    printf("Max frequency component: %f at index %d\n", maxValue, fft_index);
    
    return 0;
}
</code>
</pre>
