<h2>Union-Find (Непересекающиеся множества)</h2>

<p><strong>Категория:</strong> Структуры данных</p>

<h3>Описание</h3>
<p>Структура данных Union-Find (также называемая Disjoint Set Union - DSU) используется для хранения коллекции непересекающихся множеств. Она поддерживает две основные операции: Union (объединение двух множеств) и Find (определение, какому множеству принадлежит элемент).</p>

<h3>Принцип работы</h3>
<ol>
<li>Find(x): возвращает представителя множества, которому принадлежит x (с возможной оптимизацией сжатия пути)</li>
<li>Union(x, y): объединяет множества, содержащие x и y (с возможной оптимизацией по рангу или размеру)</li>
<li>Изначально каждый элемент является своим собственным представителем</li>
</ol>

<h3>Сложность алгоритма</h3>
<ul>
<li>Временная сложность (с оптимизациями): почти O(1) амортизированно для операций Find и Union</li>
<li>Общая сложность для m операций над n элементами: O(m α(n)), где α(n) - обратная функция Аккермана</li>
<li>Пространственная сложность: O(n)</li>
<li>С оптимизациями (сжатие пути и объединение по рангу) сложность близка к константной для практических задач</li>
</ul>

<h3>Преимущества и недостатки</h3>
<p>Преимущества:</p>
<ul>
<li>Очень высокая эффективность при работе с динамическими связными компонентами</li>
<li>Используется в алгоритме Крускала для построения минимального остовного дерева</li>
<li>Простота реализации и понимания</li>
<li>Минимальные накладные расходы на поддержание структуры</li>
</ul>

<p>Недостатки:</p>
<ul>
<li>Не поддерживает операции удаления элементов из множеств</li>
<li>Невозможно разделить объединенные множества (необратимость операций)</li>
<li>Сложный анализ временной сложности без оптимизаций</li>
<li>Ограниченный набор поддерживаемых операций</li>
</ul>

<h3>Пример использования</h3>
<pre>
<code>
class UnionFind {
  constructor(size) {
    this.parent = new Array(size);
    this.rank = new Array(size);
    
    for (let i = 0; i < size; i++) {
      this.parent[i] = i;
      this.rank[i] = 0;
    }
  }
  
  find(x) {
    if (this.parent[x] !== x) {
      // Сжатие пути
      this.parent[x] = this.find(this.parent[x]);
    }
    return this.parent[x];
  }
  
  union(x, y) {
    const rootX = this.find(x);
    const rootY = this.find(y);
    
    if (rootX === rootY) {
      return;
    }
    
    // Объединение по рангу
    if (this.rank[rootX] < this.rank[rootY]) {
      this.parent[rootX] = rootY;
    } else if (this.rank[rootX] > this.rank[rootY]) {
      this.parent[rootY] = rootX;
    } else {
      this.parent[rootY] = rootX;
      this.rank[rootX]++;
    }
  }
  
  connected(x, y) {
    return this.find(x) === this.find(y);
  }
}

// Пример использования:
const uf = new UnionFind(6);
uf.union(0, 1);
uf.union(1, 2);
uf.union(3, 4);
uf.union(4, 5);

console.log(uf.connected(0, 2)); // true
console.log(uf.connected(1, 3)); // false
console.log(uf.connected(3, 5)); // true
</code>
</pre>

<h3>Детали реализации</h3>
<ul>
<li>Используются две оптимизации: сжатие пути и объединение по рангу</li>
<li>Массив parent хранит родительские узлы для каждого элемента</li>
<li>Массив rank хранит ранг каждого дерева для оптимизации объединения</li>
<li>Операция find реализует путь сжатия для улучшения производительности</li>
<li>Общая производительность близка к константной благодаря оптимизациям</li>
</ul>
