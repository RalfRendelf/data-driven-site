<h2>Topological Sort (Топологическая сортировка)</h2>

<p><strong>Категория:</strong> Графы</p>

<h3>Описание</h3>
<p>Топологическая сортировка - это линейное расположение вершин ориентированного ациклического графа (DAG), такое что для каждого направленного ребра uv вершина u находится перед v в порядке. Используется для задач планирования, когда одни задачи должны выполняться перед другими.</p>

<h3>Принцип работы</h3>
<ol>
<li>Найти вершину без входящих ребер (степень входа равна 0)</li>
<li>Добавить эту вершину в результат и удалить её и все исходящие из неё ребра из графа</li>
<li>Повторять до тех пор, пока все вершины не будут обработаны</li>
<li>Если в графе есть цикл, топологическая сортировка невозможна</li>
</ol>

<h3>Сложность алгоритма</h3>
<ul>
<li>Временная сложность: O(V + E), где V - количество вершин, E - количество ребер. 
  - V - количество вершин, так как мы обрабатываем каждую вершину один раз
  - E - количество ребер, так как мы обрабатываем каждое ребро один раз при вычислении входящих степеней и при обработке вершин</li>
<li>Пространственная сложность: O(V)
  - O(V) для хранения массива степеней входа
  - O(V) для хранения результата
  - O(V) для очереди, которая в худшем случае может содержать все вершины
  - Итого: O(V) по памяти, так как все эти структуры пропорциональны количеству вершин</li>
</ul>

<h3>Преимущества и недостатки</h3>
<p>Преимущества:</p>
<ul>
<li>Позволяет упорядочить задачи с зависимостями</li>
<li>Полезна для определения приоритетов в выполнении операций</li>
<li>Может использоваться для проверки наличия циклов в графе</li>
</ul>

<p>Недостатки:</p>
<ul>
<li>Работает только с ориентированными ациклическими графами (DAG)</li>
<li>Результат может быть не уникальным (несколько допустимых порядков)</li>
<li>Требует предварительного анализа зависимостей</li>
</ul>

<h3>Пример использования</h3>
<pre>
<code>
function topologicalSort(graph) {
  const inDegree = {}; // количество входящих рёбер для каждой вершины
  const queue = [];
  const result = [];
  
  // Инициализация степеней входа
  for (const node in graph) {
    inDegree[node] = 0;
  }
  
  // Подсчет входящих рёбер
  for (const node in graph) {
    for (const neighbor of graph[node]) {
      inDegree[neighbor]++;
    }
  }
  
  // Добавляем вершины с входной степенью 0 в очередь
  for (const node in inDegree) {
    if (inDegree[node] === 0) {
      queue.push(node);
    }
  }
  
  // Процесс топологической сортировки
  while (queue.length > 0) {
    const node = queue.shift();
    result.push(node);
    
    // Уменьшаем входную степень соседей
    for (const neighbor of graph[node]) {
      inDegree[neighbor]--;
      
      // Если входная степень стала 0, добавляем в очередь
      if (inDegree[neighbor] === 0) {
        queue.push(neighbor);
      }
    }
  }
  
  // Проверяем, есть ли цикл
  if (result.length !== Object.keys(graph).length) {
    throw new Error('Граф содержит цикл, топологическая сортировка невозможна');
  }
  
  return result;
}

// Альтернативный метод с использованием DFS
function topologicalSortDFS(graph) {
  const visited = new Set();
  const tempVisited = new Set();
  const result = [];
  
  function dfs(node) {
    if (tempVisited.has(node)) {
      throw new Error('Граф содержит цикл');
    }
    
    if (visited.has(node)) return;
    
    tempVisited.add(node);
    
    for (const neighbor of graph[node]) {
      dfs(neighbor);
    }
    
    tempVisited.delete(node);
    visited.add(node);
    result.unshift(node); // добавляем в начало результата
  }
  
  for (const node in graph) {
    if (!visited.has(node)) {
      dfs(node);
    }
  }
  
  return result;
}

// Пример использования:
const graph = {
  'A': ['C'],
  'B': ['C', 'D'],
  'C': ['E'],
  'D': ['F'],
  'E': ['F'],
  'F': []
};

console.log(topologicalSort(graph)); // ['A', 'B', 'C', 'E', 'D', 'F'] или другой допустимый порядок
