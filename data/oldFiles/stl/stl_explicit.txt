<h3>Ключевое слово explicit в C++</h3>
<p>Ключевое слово <code>explicit</code> используется в C++ для предотвращения неявных преобразований конструкторов и функций преобразования типов.</p>

<h4>Проблема неявных преобразований</h4>
<p>Когда у класса есть конструктор, принимающий один аргумент, компилятор может использовать его для неявного преобразования типа. Это может привести к нежелательным последствиям:</p>

<pre><code>class MyClass {
public:
    MyClass(int x) { /* ... */ }  // Неявный конструктор
};

void func(MyClass obj) { /* ... */ }

func(42);  // Компилятор неявно создаст MyClass(42)
</code></pre>

<h4>Решение с explicit</h4>
<p>Использование <code>explicit</code> запрещает неявные преобразования:</p>

<pre><code>class MyClass {
public:
    explicit MyClass(int x) { /* ... */ }  // Только явные вызовы
};

void func(MyClass obj) { /* ... */ }

// func(42);  // ОШИБКА: неявное преобразование запрещено
func(MyClass(42));  // OK: явное создание объекта
func(static_cast&lt;MyClass&gt;(42));  // OK: явное преобразование
</code></pre>

<h4>Преимущества использования explicit</h4>
<ul>
<li>Предотвращает случайные и нежелательные преобразования типов</li>
<li>Повышает безопасность кода и уменьшает количество ошибок</li>
<li>Делает код более понятным и предсказуемым</li>
</ul>

<h4>Примеры использования</h4>
<p>Часто используется в стандартной библиотеке C++:</p>

<pre><code>std::vector&lt;int&gt; v(10);      // Создать вектор с 10 элементами
// std::vector&lt;int&gt; v = 10;  // Такое присваивание невозможно благодаря explicit

std::thread t([](){ /* ... */ });  
// std::thread t = [](){ /* ... */ };  // Ошибка, если бы не explicit
</code></pre>

<h4>Временная и пространственная сложность</h4>
<p>Ключевое слово <code>explicit</code> влияет только на этап компиляции и не влияет на временную или пространственную сложность во время выполнения программы.</p>
