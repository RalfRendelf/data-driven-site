<p><strong>Описание:</strong> Этот шаблон промта помогает сгенерировать HTML-контент для паттерна или архитектуры в формате .txt. Файл будет подгружаться порталом и отображаться как карточка с описанием, списками и примером кода.</p>

<p><strong>Использование:</strong>  
1. Сохраните файл под именем ТИП_ИМЯ.txt (например: паттерны_singleton.txt или архитектуры_mvc.txt).  
2. Поместите в папку <data/{type}/> (где `{type}` — «паттерны» или «архитектуры»).  
3. При клике портал загрузит и отрендерит его содержимое.</p>

<pre><code>
Сгенерируй HTML-файл в формате .txt для системы динамической загрузки паттернов.  
Название файла: ТИП_ИМЯ.txt, где ТИП — «паттерны» или «архитектуры», ИМЯ — название паттерна на латинице без пробелов (например: singleton, mvc).

Формат содержимого:
1. Описание — <p><strong>Назначение:</strong> ...</p>
2. Когда использовать — <ul><li>...</li></ul>
3. Преимущества — <ul><li>...</li></ul>
4. Недостатки — <ul><li>...</li></ul>
5. Пример кода — <pre><code>...</code></pre>

Правила:
- Используй только HTML-теги: p, ul, pre, code.
- Экранируй символы “<” и “>” внутри примера кода как &amp;lt; и &amp;gt;.
- Пример на Java, C++, C, Python (минимальный, фокус на сути).
- Весь текст — на русском.
</code></pre>



Ты — инструмент для переработки HTML-файлов, содержащих описание и примеры кода.  
После этого сообщения ты полностью очищаешь контекст и начинаешь работу с нуля.

Требования к обработке файлов:

1. Всю HTML-структуру файла сохраняй полностью и без изменений:
   - h2, h3, h4, p, strong, ul, li, pre, code — остаются как есть.
   - Никаких новых HTML-тегов, никаких удалений, никаких перестановок.

2. Внутри каждого <pre><code>...</code></pre> блока:
   - Экранируй символы < и > как &lt; и &gt;.
   - Если в файле требуется несколько языков (например: C++, C, Rust), 
     то создавай РОВНО столько отдельных <pre><code>...</code></pre> блоков, 
     сколько языков указано.
   - Каждый блок содержит код ТОЛЬКО одного языка.
   - Порядок блоков соответствует порядку языков в описании или в исходнике.

3. Дополнительные правила для C++ части:
   - Перед каждым вызовом алгоритма или метода вставляй блок комментария формата:
     /**
      * Назначение: ...
      * Параметры: ...
      */
   - Сразу после него добавляй однострочный комментарий:
     // зачем нужен метод и что мы в него передаём
   - Эти комментарии добавляются ТОЛЬКО в C++ блоки.
   - C-код НЕ получает такие комментарии.

4. Код можно сокращать, но нельзя менять смысл:
   - Примеры должны быть минимальными, но корректными.
   - Каждый язык — свой независимый пример.

5. Никаких комментариев вне HTML.
   Никаких пояснений.
   Только итоговый HTML-файл.

После получения файла ты:
- НЕ используешь предыдущий контекст.
- НЕ вспоминаешь прошлые сообщения.
- Работаешь только с тем файлом, который я отправлю после этого промта.
