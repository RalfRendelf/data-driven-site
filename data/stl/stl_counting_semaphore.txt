<h2>std::counting_semaphore</h2>

<p><strong>Категория:</strong> Многопоточность (Механизмы синхронизации)</p>

<h3>Описание</h3>
<p>std::counting_semaphore — это легковесный примитив синхронизации, который управляет внутренним счетчиком. Потоки могут уменьшать счетчик (захватывать ресурс) и увеличивать его (освобождать). Если счетчик равен нулю, поток, пытающийся его уменьшить, блокируется. В отличие от мьютекса, семафор не имеет понятия «потока-владельца».</p>

<h3>Добавлен в стандарт:</h3>
<p>C++20</p>

<h3>Основные методы:</h3>
<p>Ниже представлены ключевые методы управления семафором и пример их логики:</p>

<h4>C++</h4>
<pre>
<code>

#include &lt;semaphore&gt
#include &lt;chrono&gt

// Инициализация: разрешаем одновременно 3 потока
std::counting_semaphore<10> sem(3); 

// 1. acquire() — уменьшает счетчик. 
// Если счетчик 0, поток засыпает до появления "слота".
sem.acquire(); 

// [ Критическая секция или доступ к ограниченному ресурсу ]

// 2. release(n) — увеличивает счетчик на n (по умолчанию 1).
// Разблокирует ожидающие потоки.
sem.release(); 

// 3. try_acquire() — пытается захватить ресурс без блокировки.
// Возвращает true, если удалось, и false, если счетчик был 0.
if (sem.try_acquire()) {
    // Ресурс захвачен
    sem.release();
}

// 4. try_acquire_for() — ждет свободный слот в течение указанного времени.
using namespace std::chrono_literals;
if (sem.try_acquire_for(100ms)) {
    // Успели захватить за 100 миллисекунд
    sem.release();
}
</code>
</pre>
