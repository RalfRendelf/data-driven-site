<h2>std::binary_semaphore</h2>

<p><strong>Категория:</strong> Многопоточность (Механизмы сигнализации)</p>

<h3>Описание</h3>
<p>std::binary_semaphore — это специализация std::counting_semaphore со счетчиком, который может принимать только значения 0 и 1. Он идеально подходит для сигнализации между потоками: один поток ждет (acquire), а другой уведомляет о завершении события (release). В отличие от мьютекса, разблокировать бинарный семафор может не только тот поток, который его захватил.</p>

<h3>Добавлен в стандарт:</h3>
<p>C++20</p>

<h3>Основные методы:</h3>
<p>В этом разделе представлены основные методы std::binary_semaphore для управления доступом и синхронизации:</p>

<h4>C++</h4>
<pre>
<code>
#include &lt;semaphore&gt;
#include &lt;chrono&gt;
#include &lt;thread&gt;

// Создаем семафор, изначально закрытый (0)
std::binary_semaphore signal(0);

// Поток-отправитель:
// Выполняет работу и "открывает" семафор
signal.release(); // Увеличивает счетчик до 1, разрешая проход

// Поток-получатель:
// Блокируется до тех пор, пока счетчик не станет > 0
signal.acquire(); // Уменьшает счетчик обратно до 0 и проходит дальше

// try_acquire():
// Пытается захватить семафор немедленно. Возвращает true/false.
if (signal.try_acquire()) {
    // Ресурс был свободен, мы его захватили
    signal.release();
}

// try_acquire_until():
// Ждет захвата до определенного момента времени (time_point)
auto timeout = std::chrono::steady_clock::now() + std::chrono::seconds(1);
if (signal.try_acquire_until(timeout)) {
    // Успели захватить до наступления таймаута
    signal.release();
}
</code>
</pre>
