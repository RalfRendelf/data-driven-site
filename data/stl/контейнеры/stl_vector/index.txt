<h2>std::vector</h2>

<p><strong>Категория:</strong> Контейнеры</p>

<h3>Описание</h3>
<p>std::vector - это последовательный контейнер, который представляет собой динамический массив. Он обеспечивает возможность хранения элементов в виде массива с возможностью автоматического изменения размера при добавлении или удалении элементов.</p>

<h3>Добавлен в стандарт:</h3>
<p>C++98</p>

<h3>Основные методы:</h3>
<ul>
<li><strong>vector&lt;T&gt;::vector()</strong> - конструктор по умолчанию</li>
<li><strong>vector&lt;T&gt;::push_back(const T&amp; x)</strong> - добавляет элемент в конец вектора</li>
<li><strong>vector&lt;T&gt;::pop_back()</strong> - удаляет последний элемент вектора</li>
<li><strong>vector&lt;T&gt;::size()</strong> - возвращает количество элементов в векторе</li>
<li><strong>vector&lt;T&gt;::empty()</strong> - проверяет, пуст ли вектор</li>
<li><strong>vector&lt;T&gt;::at(size_t pos)</strong> - доступ к элементу по указанному индексу с проверкой границ</li>
<li><strong>vector&lt;T&gt;::operator[]</strong> - доступ к элементу по указанному индексу без проверки границ</li>
<li><strong>vector&lt;T&gt;::front()</strong> - доступ к первому элементу</li>
<li><strong>vector&lt;T&gt;::back()</strong> - доступ к последнему элементу</li>
<li><strong>vector&lt;T&gt;::clear()</strong> - удаляет все элементы вектора</li>
<li><strong>vector&lt;T&gt;::insert()</strong> - вставляет элементы в заданную позицию</li>
<li><strong>vector&lt;T&gt;::erase()</strong> - удаляет элементы в заданной позиции или диапазоне</li>
<li><strong>vector&lt;T&gt;::resize(size_t n)</strong> - изменяет размер вектора до указанного</li>
<li><strong>vector&lt;T&gt;::reserve(size_t n)</strong> - заранее выделяет память под n элементов</li>
</ul>

<h3>Особенности реализации:</h3>
<p>Вектор реализован как динамический массив, который автоматически увеличивает свой размер при необходимости. При добавлении элементов в вектор, когда текущей емкости недостаточно, происходит выделение новой памяти и копирование/перемещение старых элементов в новое место. Это может быть затратной операцией, поэтому рекомендуется использовать метод reserve(), если заранее известно примерное количество элементов.</p>

<h3>Временная сложность операций:</h3>
<ul>
<li>Доступ к элементу по индексу: O(1)</li>
<li>Добавление элемента в конец: O(1) амортизированное</li>
<li>Удаление элемента из конца: O(1)</li>
<li>Вставка/удаление в произвольной позиции: O(n)</li>
<li>Поиск элемента: O(n)</li>
</ul>
