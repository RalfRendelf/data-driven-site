<h2>Память и загрузка</h2>

<p><strong>Категория:</strong> Основы</p>

<h3>Описание</h3>
<p>Память и загрузка прошивки являются фундаментальными аспектами работы микроконтроллеров. Понимание структуры памяти, процесса загрузки и организации кода имеет критическое значение для эффективной разработки встраиваемых приложений.</p>

<h3>Типы памяти:</h3>
<ul>
<li><strong>Flash</strong> - энергонезависимая память для хранения программного кода и констант</li>
<li><strong>SRAM</strong> - статическая оперативная память для переменных и стека</li>
<li><strong>CCM (Core Coupled Memory)</strong> - высокоскоростная память, связанная с ядром процессора</li>
<li><strong>TCM (Tightly Coupled Memory)</strong> - память с гарантированным временем доступа, для критических данных</li>
</ul>

<h3>Карта памяти (Memory Map):</h3>
<p>Карта памяти определяет распределение адресного пространства микроконтроллера. Обычно Flash располагается в нижней части адресного пространства (0x00000000), а SRAM выше. Для Cortex-M:</p>
<ul>
<li>0x00000000 - 0x1FFFFFFF: Flash или ROM</li>
<li>0x20000000 - 0x3FFFFFFF: SRAM</li>
<li>0x40000000 - 0x5FFFFFFF: Периферия</li>
<li>0xE0000000 - 0xFFFFFFFF: Регистры ядра</li>
</ul>

<h3>Startup-файл (startup.s):</h3>
<p>Файл на языке ассемблера, содержащий векторы прерываний и начальный код инициализации:</p>

<pre>
<code>
; Файл: startup_stm32f407xx.s
; Векторы прерываний и начальная точка входа

; Вектор прерываний - начальный стек и точка сброса
Vectors    DCD    __initial_sp               ; 0x00 Вершина стека
           DCD    Reset_Handler              ; 0x04 Обработчик сброса
           DCD    NMI_Handler                ; 0x08 NMI
           DCD    HardFault_Handler          ; 0x0C Жесткий отказ
           DCD    MemManage_Handler          ; 0x10 Управление памятью
           DCD    BusFault_Handler           ; 0x14 Ошибка шины
           DCD    UsageFault_Handler         ; 0x18 Ошибка использования
           ; ... остальные векторы прерываний ...
           
; Определение обработчика сброса
Reset_Handler:
    ; Копирование секции .data из Flash в RAM
    ldr r1, =_sdata       ; Начало данных в RAM
    ldr r2, =_edata       ; Конец данных в RAM
    ldr r0, =_sidata      ; Начало данных в Flash
    
CopyDataLoop:
    cmp r1, r2
    itt lt
    ldrlt r3, [r0], #4
    strlt r3, [r1], #4
    blt CopyDataLoop
    
    ; Заполнение .bss нулями
    ldr r1, =_sbss
    ldr r2, =_ebss
    movs r3, #0
    
ZeroBSSLoop:
    cmp r1, r2
    itt lt
    strlt r3, [r1], #4
    blt ZeroBSSLoop
    
    ; Вызов функции main
    bl main
    ; Если main возвращает управление, зациклиться здесь
Hang:
    b Hang
</code>
</pre>

<h3>Reset Handler:</h3>
<p>Обработчик сброса - первая исполняемая после включения кодовая последовательность. Он инициализирует стек, копирует данные из Flash в RAM и вызывает функцию main().</p>

<h3>Линкерный скрипт (.ld):</h3>
<p>Файл, определяющий, как секции программы размещаются в физической памяти:</p>

<pre>
<code>
/* Линкерный скрипт для STM32F407VGTx */
MEMORY
{
  FLASH (rx) : ORIGIN = 0x08000000, LENGTH = 1024K
  RAM (xrw) : ORIGIN = 0x20000000, LENGTH = 192K
  CCMRAM (xrw) : ORIGIN = 0x10000000, LENGTH = 64K
}

SECTIONS
{
  .isr_vector :
  {
    . = ALIGN(4);
    KEEP(*(.isr_vector))
    . = ALIGN(4);
  } >FLASH

  .text :
  {
    . = ALIGN(4);
    *(.text)
    *(.text*)
    *(.glue_7) 
    *(.glue_7t)
    . = ALIGN(4);
    _etext = .;
  } >FLASH

  .rodata :
  {
    . = ALIGN(4);
    *(.rodata)
    *(.rodata*)
    . = ALIGN(4);
  } >FLASH
  
  .data : 
  {
    . = ALIGN(4);
    _sdata = .;
    *(.data)
    *(.data*)
    . = ALIGN(4);
    _edata = .;
    _sidata = LOADADDR(.data);
  } >RAM AT> FLASH
  
  .bss :
  {
    . = ALIGN(4);
    _sbss = .;
    __bss_start__ = .;
    *(.bss)
    *(.bss*)
    *(COMMON)
    . = ALIGN(4);
    _ebss = .;
    __bss_end__ = .;
  } >RAM
}
</code>
</pre>

<h3>Стек и куча:</h3>
<ul>
<li><strong>Стек</strong> - LIFO структура для хранения локальных переменных и адресов возврата из функций</li>
<li><strong>Куча</strong> - область памяти для динамического выделения (malloc/free)</li>
<li>Стек начинается с верхней границы RAM и растет вниз</li>
<li>Куча начинается после .bss секции и растет вверх</li>
<li>Необходимо избегать столкновения стека и кучи</li>
</ul>

<h3>Bootloader / загрузка прошивки:</h3>
<p>Bootloader - специальная программа, которая позволяет загружать другие программы во Flash. Существуют несколько способов загрузки:</p>
<ul>
<li>Через SWD/JTAG программаторы (например, ST-Link, J-Link)</li>
<li>Через UART (serial bootloader)</li>
<li>USB DFU (Device Firmware Update)</li>
<li>OTA (Over-The-Air) обновления</li>
</ul>

<h3>Векторы прерываний:</h3>
<p>Таблица адресов обработчиков прерываний, размещенная в начале Flash памяти. При возникновении прерывания процессор обращается к соответствующему элементу таблицы и переходит по указанному адресу. Для ARM Cortex-M таблица выглядит так:</p>

<pre>
<code>
; Векторы прерываний для STM32F4
; Смещение  Адрес в Flash    Назначение
; 0x00      0x08000000      Вершина стека (SP)
; 0x04      0x08000004      Reset Handler
; 0x08      0x08000008      NMI Handler
; 0x0C      0x0800000C      Hard Fault Handler
; 0x10      0x08000010      Memory Management Fault
; ...
; 0x3C      0x0800003C      SVCall Handler
; 0x40      0x08000040      Debug Monitor Handler
; 0x44      0x08000044      PendSV Handler
; 0x48      0x08000048      SysTick Handler
</code>
</pre>

<h3>Рекомендации по оптимизации памяти:</h3>
<ul>
<li>Использовать CCM или TCM для критических участков кода и данных</li>
<li>Размещать часто используемые данные в быстрой памяти</li>
<li>Контролировать размер стека и кучи</li>
<li>Использовать .bss секцию для инициализированных нулем переменных</li>
</ul>
