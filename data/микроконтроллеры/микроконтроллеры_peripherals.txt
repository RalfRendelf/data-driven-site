<h2>Периферия микроконтроллеров</h2>

<p><strong>Категория:</strong> Периферия</p>

<h3>Описание</h3>
<p>Периферия микроконтроллера представляет собой встроенные модули, которые позволяют взаимодействовать с внешними устройствами и выполнять различные функции. Эти модули реализованы в виде аппаратных блоков, интегрированных в чип микроконтроллера. Периферия может быть разделена на базовые и продвинутые компоненты.</p>

<h3>Базовая периферия:</h3>

<h4>GPIO (General Purpose Input/Output)</h4>
<p>Универсальные порты ввода-вывода, позволяющие управлять цифровыми сигналами. Подробнее см. в файле <a href="микроконтроллеры_gpio.txt">микроконтроллеры_gpio.txt</a>.</p>

<h4>UART (Universal Asynchronous Receiver/Transmitter)</h4>
<p>Последовательный асинхронный интерфейс передачи данных. Используется для связи с другими устройствами, отладки и передачи данных. Обычно используется для подключения беспроводных модулей, датчиков и других микроконтроллеров.</p>

<h4>SPI (Serial Peripheral Interface)</h4>
<p>Синхронный последовательный интерфейс, используемый для связи с внешними устройствами. Использует три провода (CLK, MOSI, MISO) и отдельный сигнал CS для каждого устройства. Обладает высокой скоростью передачи данных.</p>

<h4>I2C (Inter-Integrated Circuit)</h4>
<p>Двухпроводной последовательный интерфейс, используемый для связи с множеством периферийных устройств. Использует линии SDA (данные) и SCL (тактовый сигнал). Позволяет подключать множество устройств с уникальными адресами.</p>

<p>Более подробную информацию о UART, SPI и I2C см. в файле <a href="микроконтроллеры_serial_interfaces.txt">микроконтроллеры_serial_interfaces.txt</a>.</p>

<h4>ADC (Analog-to-Digital Converter)</h4>
<p>Аналого-цифровой преобразователь, преобразующий аналоговые сигналы во входных датчиках в цифровые значения. Позволяет измерять напряжение с высокой точностью.</p>

<h4>DAC (Digital-to-Analog Converter)</h4>
<p>Цифро-аналоговый преобразователь, создающий аналоговые сигналы по цифровым данным. Используется для генерации эталонных напряжений и сигналов.</p>

<p>Более подробную информацию о ADC и DAC см. в файле <a href="микроконтроллеры_adc_dac.txt">микроконтроллеры_adc_dac.txt</a>.</p>

<h4>PWM / Timers</h4>
<p>Широтно-импульсная модуляция и таймеры используются для генерации сигналов с заданным коэффициентом заполнения и точного измерения времени. Подробнее см. в файле <a href="микроконтроллеры_timers.txt">микроконтроллеры_timers.txt</a>.</p>

<h4>Watchdog (IWDG/WWDG)</h4>
<p>Сторожевые таймеры, используемые для восстановления системы при зависании программного обеспечения. Внутренний сторожевой таймер (IWDG) и оконный сторожевой таймер (WWDG) обеспечивают надежность системы.</p>

<h4>RTC (Real-Time Clock)</h4>
<p>Модуль реального времени, который продолжает работать даже при отключенном основном питании (с резервным питанием). Обеспечивает точное время и дату.</p>

<h3>Продвинутая периферия:</h3>

<h4>DMA (Direct Memory Access)</h4>
<p>Прямой доступ к памяти позволяет периферийным устройствам напрямую взаимодействовать с памятью, минуя CPU. Это значительно повышает эффективность обработки больших объемов данных.</p>

<h4>USB (CDC/HID/MSC)</h4>
<p>Универсальная последовательная шина, позволяющая подключать множество устройств. Поддерживает различные классы устройств, такие как CDC (устройства управления коммуникациями), HID (устройства взаимодействия с человеком), MSC (устройства хранения массовой информации).</p>

<h4>CAN / CAN FD (Controller Area Network)</h4>
<p>Протокол передачи данных, широко используемый в автомобильной промышленности и промышленных приложениях. Обеспечивает надежную передачу данных в условиях электромагнитных помех.</p>

<h4>Ethernet (MAC/PHY)</h4>
<p>Сетевой интерфейс для подключения к локальным сетям и интернету. Включает MAC (Media Access Control) и PHY (Physical Layer) компоненты.</p>

<h4>SDIO / SDMMC (Secure Digital Input/Output / Secure Digital MultiMedia Card)</h4>
<p>Интерфейсы для подключения карт памяти SD/MMC. Позволяют расширять хранилище данных в микроконтроллере.</p>

<h4>QSPI / OctoSPI</h4>
<p>Высокоскоростные последовательные интерфейсы, используемые для подключения флэш-памяти и других периферийных устройств. OctoSPI позволяет использовать до 8 линий данных для увеличения скорости передачи.</p>

<h4>LCD / LTDC (LCD-TFT Display Controller)</h4>
<p>Контроллеры дисплея, используемые для управления жидкокристаллическими экранами. LTDC обеспечивает прямой вывод графики на дисплей без участия CPU.</p>

<h4>Touch Controllers</h4>
<p>Контроллеры сенсорных экранов, обеспечивающие обработку касаний и жестов на сенсорных дисплеях.</p>

<h3>Примеры использования периферии:</h3>

<h4>Пример инициализации нескольких периферийных модулей:</h4>

<pre>
<code>
#include "stm32f4xx_hal.h"

// Глобальные переменные для периферии
UART_HandleTypeDef huart2;
I2C_HandleTypeDef hi2c1;
ADC_HandleTypeDef hadc1;
TIM_HandleTypeDef htim2;

void SystemPeripherals_Init(void) {
    // Включить тактирование периферии
    __HAL_RCC_GPIOA_CLK_ENABLE();
    __HAL_RCC_GPIOB_CLK_ENABLE();
    __HAL_RCC_USART2_CLK_ENABLE();
    __HAL_RCC_I2C1_CLK_ENABLE();
    __HAL_RCC_ADC1_CLK_ENABLE();
    __HAL_RCC_TIM2_CLK_ENABLE();

    // Настроить GPIO для UART
    GPIO_InitTypeDef gpiob_init = {0};
    gpiob_init.Pin = GPIO_PIN_6 | GPIO_PIN_7;  // TX:PB6, RX:PB7
    gpiob_init.Mode = GPIO_MODE_AF_PP;
    gpiob_init.Pull = GPIO_NOPULL;
    gpiob_init.Speed = GPIO_SPEED_FREQ_VERY_HIGH;
    gpiob_init.Alternate = GPIO_AF7_USART2;
    HAL_GPIO_Init(GPIOB, &gpiob_init);

    // Настроить параметры UART
    huart2.Instance = USART2;
    huart2.Init.BaudRate = 115200;
    huart2.Init.WordLength = UART_WORDLENGTH_8B;
    huart2.Init.StopBits = UART_STOPBITS_1;
    huart2.Init.Parity = UART_PARITY_NONE;
    huart2.Init.Mode = UART_MODE_TX_RX;
    huart2.Init.HwFlowCtl = UART_HWCONTROL_NONE;
    huart2.Init.OverSampling = UART_OVERSAMPLING_16;
    if (HAL_UART_Init(&huart2) != HAL_OK) {
        Error_Handler();
    }

    // Настроить GPIO для I2C
    GPIO_InitTypeDef gpioa_init = {0};
    gpioa_init.Pin = GPIO_PIN_9 | GPIO_PIN_10;  // SCL:PA9, SDA:PA10
    gpioa_init.Mode = GPIO_MODE_AF_OD;
    gpioa_init.Pull = GPIO_PULLUP;
    gpioa_init.Speed = GPIO_SPEED_FREQ_VERY_HIGH;
    HAL_GPIO_Init(GPIOA, &gpioa_init);

    // Настроить параметры I2C
    hi2c1.Instance = I2C1;
    hi2c1.Init.ClockSpeed = 100000;
    hi2c1.Init.DutyCycle = I2C_DUTYCYCLE_2;
    hi2c1.Init.OwnAddress1 = 0;
    hi2c1.Init.AddressingMode = I2C_ADDRESSINGMODE_7BIT;
    hi2c1.Init.DualAddressMode = I2C_DUALADDRESS_DISABLE;
    hi2c1.Init.OwnAddress2 = 0;
    hi2c1.Init.GeneralCallMode = I2C_GENERALCALL_DISABLE;
    hi2c1.Init.NoStretchMode = I2C_NOSTRETCH_DISABLE;
    if (HAL_I2C_Init(&hi2c1) != HAL_OK) {
        Error_Handler();
    }

    // Настроить ADC
    ADC_ChannelConfTypeDef sConfig = {0};
    hadc1.Instance = ADC1;
    hadc1.Init.ClockPrescaler = ADC_CLOCK_SYNC_PCLK_DIV4;
    hadc1.Init.Resolution = ADC_RESOLUTION_12B;
    hadc1.Init.ScanConvMode = DISABLE;
    hadc1.Init.ContinuousConvMode = DISABLE;
    hadc1.Init.DiscontinuousConvMode = DISABLE;
    hadc1.Init.ExternalTrigConvEdge = ADC_EXTERNALTRIGCONVEDGE_NONE;
    hadc1.Init.ExternalTrigConv = ADC_SOFTWARE_START;
    hadc1.Init.DataAlign = ADC_DATAALIGN_RIGHT;
    hadc1.Init.NbrOfConversion = 1;
    hadc1.Init.DMAContinuousRequests = DISABLE;
    hadc1.Init.EOCSelection = ADC_EOC_SINGLE_CONV;
    if (HAL_ADC_Init(&hadc1) != HAL_OK) {
        Error_Handler();
    }

    // Настроить канал ADC
    sConfig.Channel = ADC_CHANNEL_TEMPSENSOR;
    sConfig.Rank = 1;
    sConfig.SamplingTime = ADC_SAMPLETIME_480CYCLES;
    if (HAL_ADC_ConfigChannel(&hadc1, &sConfig) != HAL_OK) {
        Error_Handler();
    }

    // Настроить таймер для PWM
    htim2.Instance = TIM2;
    htim2.Init.Prescaler = 8400 - 1;  // Предполагаем 84MHz / 8400 = 10kHz
    htim2.Init.CounterMode = TIM_COUNTERMODE_UP;
    htim2.Init.Period = 1000 - 1;    // 10kHz / 1000 = 10Hz
    htim2.Init.ClockDivision = TIM_CLOCKDIVISION_DIV1;
    htim2.Init.AutoReloadPreload = TIM_AUTORELOAD_PRELOAD_DISABLE;
    if (HAL_TIM_Base_Init(&htim2) != HAL_OK) {
        Error_Handler();
    }
}

int main(void) {
    HAL_Init();
    SystemClock_Config();
    SystemPeripherals_Init();

    while(1) {
        // Чтение температуры с помощью встроенного сенсора
        HAL_ADC_Start(&hadc1);
        if (HAL_ADC_PollForConversion(&hadc1, 100) == HAL_OK) {
            uint32_t adc_value = HAL_ADC_GetValue(&hadc1);
            // Преобразовать в температуру
            float temperature = ((float)(1.43 - (3.3 * adc_value / 4096))) / 0.0043 + 25.0;
            
            char buffer[50];
            sprintf(buffer, "Temperature: %.2f°C\r\n", temperature);
            HAL_UART_Transmit(&huart2, (uint8_t*)buffer, strlen(buffer), HAL_MAX_DELAY);
        }
        HAL_ADC_Stop(&hadc1);

        // Запустить таймер на 1 секунду
        HAL_TIM_Base_Start(&htim2);
        HAL_Delay(1000);
        HAL_TIM_Base_Stop(&htim2);

        // Пример общения с I2C устройством (адрес 0x18)
        uint8_t data_to_send = 0xAB;
        HAL_I2C_Master_Transmit(&hi2c1, (uint16_t)(0x18<<1), &data_to_send, 1, 100);
    }
}
</code>
</pre>

<h3>Рекомендации по использованию периферии:</h3>
<ul>
<li>Используйте DMA для высокоскоростных периферийных устройств, чтобы освободить CPU</li>
<li>Всегда проверяйте статус операций с периферией</li>
<li>Планируйте использование пинов, чтобы избежать конфликта функций</li>
<li>Используйте прерывания для асинхронных операций</li>
<li>Правильно управляйте тактированием периферии для экономии энергии</li>
</ul>
