<h2>Права доступа в Linux: rwx, chmod, chown, umask</h2>

<p>Модель прав доступа в Linux основана на пользователях, группах и наборах разрешений. Каждый файл и каталог имеет владельца, группу и набор прав, определяющих, кто может читать, изменять или выполнять объект. Понимание механики rwx, chmod, chown и umask необходимо для системного программирования, DevOps и администрирования.</p>

<h3>Теоретические основы</h3>

<p>Каждый файл в Linux имеет три набора прав:</p>

<ul>
  <li><strong>user (u)</strong> — владелец файла;</li>
  <li><strong>group (g)</strong> — группа файла;</li>
  <li><strong>others (o)</strong> — все остальные пользователи.</li>
</ul>

<p>Каждый набор содержит три бита:</p>

<ul>
  <li><strong>r</strong> — read (чтение);</li>
  <li><strong>w</strong> — write (запись);</li>
  <li><strong>x</strong> — execute (выполнение).</li>
</ul>

<p>Права отображаются в виде строки:</p>

<pre><code>
-rwxr-x---
</code></pre>

<p>Где:</p>

<ul>
  <li>первый символ — тип файла (<code>-</code> — обычный файл, <code>d</code> — каталог);</li>
  <li>следующие 3 символа — права владельца;</li>
  <li>следующие 3 — группы;</li>
  <li>последние 3 — остальных пользователей.</li>
</ul>

<h3>Числовое представление прав</h3>

<p>Каждый бит имеет числовое значение:</p>

<ul>
  <li>r = 4</li>
  <li>w = 2</li>
  <li>x = 1</li>
</ul>

<p>Например:</p>

<ul>
  <li><code>7</code> = rwx (4+2+1)</li>
  <li><code>5</code> = r-x (4+0+1)</li>
  <li><code>6</code> = rw- (4+2+0)</li>
</ul>

<p>Комбинация прав записывается как три числа: <code>u g o</code>.</p>

<p>Например:</p>

<pre><code>
chmod 750 file
</code></pre>

<p>Означает:</p>

<ul>
  <li>владелец: rwx (7)</li>
  <li>группа: r-x (5)</li>
  <li>остальные: --- (0)</li>
</ul>

<h3>Команда chmod</h3>

<p><strong>chmod</strong> изменяет права доступа.</p>

<h4>Числовой формат</h4>

<pre><code>
chmod 644 file
chmod 755 script.sh
chmod 700 secret.txt
</code></pre>

<h4>Символьный формат</h4>

<pre><code>
chmod u+x script.sh
chmod g-w file
chmod o+r file
chmod ug+rwx dir
</code></pre>

<h3>Команда chown</h3>

<p><strong>chown</strong> изменяет владельца и группу файла.</p>

<pre><code>
chown user file
chown user:group file
chown :group file
chown -R user:group dir/
</code></pre>

<h3>Команда chgrp</h3>

<p>Меняет только группу:</p>

<pre><code>
chgrp developers file
</code></pre>

<h3>umask — маска создания файлов</h3>

<p><strong>umask</strong> определяет, какие права будут <strong>отключены</strong> при создании новых файлов и каталогов.</p>

<p>Базовые права по умолчанию:</p>

<ul>
  <li>файлы: 666 (rw-rw-rw-)</li>
  <li>каталоги: 777 (rwxrwxrwx)</li>
</ul>

<p>umask вычитает биты:</p>

<pre><code>
umask 022
</code></pre>

<p>Результат:</p>

<ul>
  <li>файлы: 644</li>
  <li>каталоги: 755</li>
</ul>

<h4>Пример</h4>

<pre><code>
umask_example.txt
</code></pre>

<h3>Примеры использования</h3>

<h4>Пример: проверка прав файла</h4>

<pre><code>
permissions_ls_example.txt
</code></pre>

---

<h4>Пример: установка прав для скрипта</h4>

<pre><code>

</code></pre>

---

<h4>Пример: смена владельца каталога</h4>

<pre><code>

</code></pre>

---

<h4>Пример: создание каталога с ограниченными правами</h4>

<pre><code>

</code></pre>

---

<h3>Практические применения</h3>

<ul>
  <li>ограничение доступа к конфигурациям и секретам;</li>
  <li>настройка прав веб‑серверов (nginx, apache);</li>
  <li>создание приватных каталогов пользователей;</li>
  <li>управление доступом к логам и сервисам;</li>
  <li>настройка umask в системных сервисах;</li>
  <li>обеспечение безопасности скриптов и бинарников.</li>
</ul>

<h3>Особенности и подводные камни</h3>

<ul>
  <li>chmod 777 почти всегда ошибка — избегайте полного доступа.</li>
  <li>umask влияет только на новые файлы, не на существующие.</li>
  <li>root может игнорировать многие ограничения.</li>
  <li>права каталогов отличаются от прав файлов (x = вход в каталог).</li>
  <li>неправильные права на /home могут нарушить безопасность.</li>
  <li>не используйте chown без понимания — можно сломать систему.</li>
</ul>

<h3>Заключение</h3>

<p>Права доступа — ключевой механизм безопасности Linux. Понимание rwx, chmod, chown и umask позволяет правильно настраивать доступ к файлам, каталогам и сервисам. Эти знания необходимы для системных программистов, DevOps и администраторов, работающих с Linux на уровне системы.</p>
