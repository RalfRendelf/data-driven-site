<h2>Управление беспроводными сетями в Linux: iw, iwconfig, wpa_supplicant</h2>

<p>
Беспроводные сети (Wi-Fi) стали неотъемлемой частью современных вычислений. В Linux управление беспроводными интерфейсами осуществляется с помощью специализированных утилит, таких как <code>iw</code>, <code>iwconfig</code>, <code>wpa_supplicant</code> и других.
В этой статье рассмотрены основные инструменты для настройки и управления Wi-Fi в Linux, а также их практические применения.
</p>

---

<h3>Теоретические основы</h3>

<h4>Что такое беспроводной интерфейс?</h4>
<p>
<strong>Беспроводной интерфейс</strong> — это сетевой интерфейс, который позволяет устройству подключаться к беспроводным сетям (например, Wi-Fi).
В Linux беспроводные интерфейсы обычно обозначаются как <code>wlan0</code>, <code>wlp3s0</code> и т. д.
</p>

<h4>Ключевые понятия</h4>
<ul>
  <li><strong>SSID (Service Set Identifier)</strong>: Имя беспроводной сети.</li>
  <li><strong>BSSID (Basic Service Set Identifier)</strong>: MAC-адрес точки доступа.</li>
  <li><strong>Канал (Channel)</strong>: Частотный канал, на котором работает сеть.</li>
  <li><strong>Тип безопасности</strong>: Метод аутентификации (Open, WEP, WPA, WPA2, WPA3).</li>
  <li><strong>Скорость передачи данных</strong>: Скорость соединения (например, 54 Мбит/с, 300 Мбит/с).</li>
  <li><strong>Сила сигнала</strong>: Уровень сигнала от точки доступа.</li>
</ul>

<h3>iwconfig: Устаревшая утилита для управления Wi-Fi</h3>

<h4>Что такое iwconfig?</h4>
<p>
<code>iwconfig</code> — это устаревшая утилита для настройки беспроводных интерфейсов в Linux.
Она позволяет:
</p>
<ul>
  <li>Просматривать информацию о беспроводных интерфейсах.</li>
  <li>Настраивать параметры беспроводной сети (SSID, канал, ключ и т. д.).</li>
  <li>Управлять режимом работы интерфейса (ad-hoc, managed и т. д.).</li>
</ul>

<h4>Синтаксис</h4>
<pre><code>
iwconfig_syntax.txt
</code></pre>

<h4>Примеры использования</h4>
<pre><code>
iwconfig_examples.sh
</code></pre>

Она предоставляет более мощные и удобные средства для работы с беспроводными сетями.
</p>

<h3>wpa_supplicant: Подключение к защищённым Wi-Fi сетям</h3>

<h4>Что такое wpa_supplicant?</h4>
<p>
<code>wpa_supplicant</code> — это утилита для подключения к беспроводным сетям с поддержкой различных методов аутентификации, включая:
</p>
<ul>
  <li><strong>WPA-Personal</strong> (WPA-PSK).</li>
  <li><strong>WPA-Enterprise</strong> (WPA-EAP).</li>
  <li><strong>WPA2-Personal</strong> (WPA2-PSK).</li>
  <li><strong>WPA2-Enterprise</strong> (WPA2-EAP).</li>
  <li><strong>WPA3</strong>.</li>
</ul>

<h4>Конфигурация wpa_supplicant</h4>
<p>
Конфигурация <code>wpa_supplicant</code> хранится в файле <code>/etc/wpa_supplicant/wpa_supplicant.conf</code>.
</p>

<h5>Пример файла конфигурации</h5>
<pre><code>
wpa_supplicant_conf_example.txt
</code></pre>

<table>
  <tr>
    <th>Параметр</th>
    <th>Описание</th>
  </tr>
  <tr>
    <td><code>ctrl_interface</code></td>
    <td>Интерфейс управления (используется для взаимодействия с <code>wpa_cli</code>).</td>
  </tr>
  <tr>
    <td><code>update_config</code></td>
    <td>Разрешает <code>wpa_supplicant</code> обновлять конфигурацию.</td>
  </tr>
  <tr>
    <td><code>country</code></td>
    <td>Страна (для соблюдения локальных регуляторных требований).</td>
  </tr>
  <tr>
    <td><code>ssid</code></td>
    <td>Имя беспроводной сети.</td>
  </tr>
  <tr>
    <td><code>psk</code></td>
    <td>Пароль для WPA-PSK.</td>
  </tr>
  <tr>
    <td><code>key_mgmt</code></td>
    <td>Тип управления ключами (NONE, WPA-PSK, WPA-EAP).</td>
  </tr>
  <tr>
    <td><code>eap</code></td>
    <td>Протокол EAP (PEAP, TLS, TTLS).</td>
  </tr>
  <tr>
    <td><code>identity</code></td>
    <td>Имя пользователя для EAP.</td>
  </tr>
  <tr>
    <td><code>password</code></td>
    <td>Пароль для EAP.</td>
  </tr>
</table>

<h4>Запуск wpa_supplicant</h4>
<p>
Для подключения к беспроводной сети с помощью <code>wpa_supplicant</code> выполните:
</p>
<pre><code>
wpa_supplicant_run.sh
</code></pre>
<p>
<strong>Опции:</strong>
</p>
<ul>
  <li><code>-B</code> — запуск в фоновом режиме.</li>
  <li><code>-i wlan0</code> — интерфейс для подключения.</li>
  <li><code>-c</code> — файл конфигурации.</li>
</ul>

<h4>Просмотр статуса подключения</h4>
<p>
Для просмотра статуса подключения используйте <code>wpa_cli</code>:
</p>
<pre><code>
wpa_cli_examples.sh
</code></pre>

<h3>Особенности и подводные камни</h3>

<h4>Проблемы с подключением к Wi-Fi</h4>
<p>
Если подключение к Wi-Fi не работает, проверьте:
</p>
<ul>
  <li>Правильность имени сети (SSID) и пароля.</li>
  <li>Тип безопасности (WPA-PSK, WPA2-PSK и т. д.).</li>
  <li>Наличие драйверов для беспроводного адаптера.</li>
  <li>Статус интерфейса (<code>ip link show wlan0</code>).</li>
</ul>
<pre><code>
check_wifi.sh
</code></pre>

<h4>Проблемы с драйверами</h4>
<p>
Если беспроводной адаптер не определяется, проверьте:
</p>
<ul>
  <li>Наличие драйвера для адаптера.</li>
  <li>Загружен ли драйвер в ядро.</li>
</ul>
<pre><code>
check_driver.sh
</code></pre>

<h4>Проблемы с точкой доступа</h4>
<p>
Если точка доступа не работает, проверьте:
</p>
<ul>
  <li>Наличие поддержки режима AP в драйвере.</li>
  <li>Конфигурацию <code>hostapd</code>.</li>
  <li>Конфигурацию <code>dnsmasq</code>.</li>
  <li>Настройки маршрутизации и NAT.</li>
</ul>
<pre><code>
check_ap.sh
</code></pre>
