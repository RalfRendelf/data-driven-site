<h2>DNS и разрешение имён в Linux: /etc/resolv.conf, systemd-resolved, dig, nslookup</h2>

<p>
<strong>DNS (Domain Name System)</strong> — это система, которая преобразует символьные имена доменов (например, <code>google.com</code>) в IP-адреса (например, <code>172.217.16.206</code>).
В Linux разрешение имён может осуществляться разными способами: через файл <code>/etc/resolv.conf</code>, сервис <code>systemd-resolved</code>, утилиты <code>dig</code>, <code>nslookup</code> и другие.
</p>

<p>
В этой статье рассмотрены:
</p>
<ul>
  <li>Основы работы DNS.</li>
  <li>Конфигурация разрешения имён в Linux.</li>
  <li>Использование утилит <code>dig</code>, <code>nslookup</code>, <code>host</code>.</li>
  <li>Настройка <code>systemd-resolved</code>.</li>
  <li>Практические примеры на C и C++ для работы с DNS.</li>
</ul>

<h3>Теоретические основы</h3>

<h4>Что такое DNS?</h4>
<p>
<strong>DNS (Domain Name System)</strong> — это распределённая система, которая обеспечивает:
</p>
<ul>
  <li><strong>Разрешение имён</strong>: Преобразование имён доменов в IP-адреса.</li>
  <li><strong>Обратное разрешение</strong>: Преобразование IP-адресов в имена доменов.</li>
  <li><strong>Распределённое хранение</strong>: Данные о доменах хранятся на множестве DNS-серверов.</li>
  <li><strong>Иерархическая структура</strong>: Домены организованы в дерево (например, <code>example.com</code>).</li>
</ul>

<h4>Типы DNS-записей</h4>
<p>
DNS-записи определяют различные типы информации о доменах:
</p>
<table>
  <tr>
    <th>Тип записи</th>
    <th>Описание</th>
    <th>Пример</th>
  </tr>
  <tr>
    <td><code>A</code></td>
    <td>Сопоставляет имя домена с IPv4-адресом</td>
    <td><code>example.com. IN A 192.0.2.1</code></td>
  </tr>
  <tr>
    <td><code>AAAA</code></td>
    <td>Сопоставляет имя домена с IPv6-адресом</td>
    <td><code>example.com. IN AAAA 2001:db8::1</code></td>
  </tr>
  <tr>
    <td><code>CNAME</code></td>
    <td>Создаёт псевдоним для домена</td>
    <td><code>www.example.com. IN CNAME example.com.</code></td>
  </tr>
  <tr>
    <td><code>MX</code></td>
    <td>Указывает почтовые серверы для домена</td>
    <td><code>example.com. IN MX 10 mail.example.com.</code></td>
  </tr>
  <tr>
    <td><code>NS</code></td>
    <td>Указывает DNS-серверы для домена</td>
    <td><code>example.com. IN NS ns1.example.com.</code></td>
  </tr>
  <tr>
    <td><code>TXT</code></td>
    <td>Хранит текстовые данные (например, для SPF, DKIM)</td>
    <td><code>example.com. IN TXT "v=spf1 include:_spf.example.com ~all"</code></td>
  </tr>
  <tr>
    <td><code>SOA</code></td>
    <td>Указывает начало зоны ответственности</td>
    <td><code>example.com. IN SOA ns1.example.com. admin.example.com. 2023010101 3600 1800 604800 86400</code></td>
  </tr>
  <tr>
    <td><code>PTR</code></td>
    <td>Сопоставляет IP-адрес с именем домена (обратный DNS)</td>
    <td><code>1.2.0.192.in-addr.arpa. IN PTR example.com.</code></td>
  </tr>
</table>

<h4>Процесс разрешения имён</h4>
<p>
Процесс разрешения имён в Linux включает несколько этапов:
</p>
<ol>
  <li><strong>Проверка кэша</strong>: Система проверяет локальный кэш (например, в <code>systemd-resolved</code>).</li>
  <li><strong>Проверка /etc/hosts</strong>: Система проверяет файл <code>/etc/hosts</code> для статических сопоставлений.</li>
  <li><strong>Обращение к DNS-серверам</strong>: Система отправляет запрос на настроенные DNS-серверы.</li>
  <li><strong>Рекурсивное разрешение</strong>: DNS-серверы рекурсивно разрешают имя, обращаясь к другим серверам.</li>
</ol>

<h4>Файл /etc/resolv.conf</h4>
<p>
Файл <code>/etc/resolv.conf</code> содержит конфигурацию DNS-клиента. Он определяет:
</p>
<ul>
  <li>Адреса DNS-серверов.</li>
  <li>Домен поиска.</li>
  <li>Параметры разрешения.</li>
</ul>
<p>
Пример файла <code>/etc/resolv.conf</code>:
</p>
<pre><code>
resolv_conf_example.txt
</code></pre>

<table>
  <tr>
    <th>Параметр</th>
    <th>Описание</th>
  </tr>
  <tr>
    <td><code>nameserver</code></td>
    <td>IP-адрес DNS-сервера.</td>
  </tr>
  <tr>
    <td><code>search</code></td>
    <td>Домен поиска для неполных имён.</td>
  </tr>
  <tr>
    <td><code>options</code></td>
    <td>Параметры разрешения (например, таймаут, количество попыток).</td>
  </tr>
</table>

<h3>systemd-resolved: Современный DNS-резолвер</h3>

<h4>Что такое systemd-resolved?</h4>
<p>
<code>systemd-resolved</code> — это сервис в системах с <code>systemd</code>, который предоставляет:
</p>
<ul>
  <li><strong>Кэширование DNS-запросов</strong>.</li>
  <li><strong>Поддержку нескольких DNS-серверов</strong>.</li>
  <li><strong>Поддержку DNS-over-TLS (DoT)</strong>.</li>
  <li><strong>Интеграцию с NetworkManager</strong>.</li>
</ul>

<h3>Особенности и подводные камни</h3>

<h4>Проблемы с кэшированием DNS</h4>
<p>
Кэширование DNS может привести к тому, что изменения в DNS-записях не будут сразу видны. Для решения:
</p>
<ul>
  <li>Очищайте кэш DNS:</li>
</ul>
<pre><code>
clear_dns_cache.sh
</code></pre>
<ul>
  <li>Уменьшайте время жизни (TTL) DNS-записей.</li>
</ul>

<h4>Проблемы с /etc/resolv.conf</h4>
<p>
Если изменения в <code>/etc/resolv.conf</code> не применяются, проверьте:
</p>
<ul>
  <li>Права доступа к файлу.</li>
  <li>Наличие символических ссылок (например, <code>/etc/resolv.conf</code> может быть ссылкой на <code>/run/systemd/resolve/resolv.conf</code>).</li>
  <li>Настройки <code>systemd-resolved</code>.</li>
</ul>

<h4>Проблемы с разрешением имён</h4>
<p>
Если имена доменов не разрешаются, проверьте:
</p>
<ul>
  <li>Настройки в <code>/etc/resolv.conf</code>.</li>
  <li>Работу DNS-серверов:</li>
</ul>
<pre><code>
test_dns.sh
</code></pre>
<ul>
  <li>Файл <code>/etc/nsswitch.conf</code>:</li>
</ul>
<pre><code>
nsswitch_conf_example.txt
</code></pre>
<p>
Если в строке <code>hosts</code> сначала указано <code>files</code>, система сначала проверит <code>/etc/hosts</code>, а затем обратится к DNS.
</p>

<h4>Проблемы с IPv6</h4>
<p>
Если IPv6-адреса не разрешаются, проверьте:
</p>
<ul>
  <li>Наличие IPv6-соединения.</li>
  <li>Настройки DNS-серверов для IPv6.</li>
</ul>
<pre><code>
test_ipv6_dns.sh
</code></pre>

В Linux существует несколько способов настройки и использования DNS: через файл <code>/etc/resolv.conf</code>, сервис <code>systemd-resolved</code>, утилиты <code>dig</code>, <code>nslookup</code> и программные интерфейсы на C и C++.
