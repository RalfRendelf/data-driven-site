<h2>Маршрутизация и сетевые интерфейсы в Linux: iproute2, таблицы маршрутизации</h2>

<p>
В Linux управление сетевыми интерфейсами и маршрутизацией осуществляется с помощью мощных инструментов, таких как <strong>iproute2</strong>.
Эти инструменты позволяют настраивать сетевые параметры, управлять маршрутами, интерфейсами и таблицами маршрутизации.
В этой статье рассмотрены основные концепции маршрутизации, работа с сетевыми интерфейсами, а также использование утилит <code>ip</code> и <code>route</code>.
</p>

---

<h3>Теоретические основы</h3>

<h4>Что такое сетевой интерфейс?</h4>
<p>
<strong>Сетевой интерфейс</strong> — это программный или аппаратный компонент, который позволяет устройству подключаться к сети.
В Linux сетевые интерфейсы могут быть:
</p>
<ul>
  <li><strong>Физические</strong> (Ethernet, Wi-Fi).</li>
  <li><strong>Виртуальные</strong> (Loopback, VPN, мосты).</li>
  <li><strong>Логические</strong> (VLAN, туннели).</li>
</ul>

<h4>Что такое маршрутизация?</h4>
<p>
<strong>Маршрутизация</strong> — это процесс определения пути, по которому пакет данных должен перемещаться от отправителя к получателю.
В Linux маршрутизация реализуется с помощью <strong>таблиц маршрутизации</strong>, которые содержат правила для пересылки пакетов.
</p>

<h4>Ключевые понятия</h4>
<ul>
  <li><strong>IP-адрес</strong>: Уникальный идентификатор устройства в сети.</li>
  <li><strong>Шлюз (Gateway)</strong>: Устройство, которое пересылает пакеты между сетями.</li>
  <li><strong>Маска подсети (Netmask)</strong>: Определяет, какая часть IP-адреса относится к сети, а какая — к хосту.</li>
  <li><strong>Метрика (Metric)</strong>: Стоимость маршрута (меньше = лучше).</li>
  <li><strong>MTU (Maximum Transmission Unit)</strong>: Максимальный размер пакета, который может быть передан без фрагментации.</li>
</ul>

<h3>Работа с сетевыми интерфейсами</h3>

<h4>Просмотр интерфейсов</h4>
<p>
Для просмотра списка сетевых интерфейсов и их параметров используйте команду <code>ip a</code> (или <code>ip addr</code>):
</p>
<pre><code>
ip_addr_show.sh
</code></pre>
<p>
Пример вывода:
</p>
<pre><code>
ip_addr_example.txt
</code></pre>

<h4>Просмотр статистики интерфейсов</h4>
<p>
Для просмотра статистики по интерфейсам (пакеты, байты, ошибки) используйте команду:
</p>
<pre><code>
ip_stats_show.sh
</code></pre>

<h4>Включение и отключение интерфейсов</h4>
<p>
Для включения или отключения сетевого интерфейса используйте команды:
</p>
<pre><code>
ip_link_up_down.sh
</code></pre>

<h4>Назначение IP-адреса</h4>
<p>
Для назначения IP-адреса интерфейсу используйте команду:
</p>
<pre><code>
ip_addr_add.sh
</code></pre>

<h4>Удаление IP-адреса</h4>
<p>
Для удаления IP-адреса с интерфейса используйте команду:
</p>
<pre><code>
ip_addr_del.sh
</code></pre>

---
<h3>Работа с таблицами маршрутизации</h3>

<h4>Просмотр таблицы маршрутизации</h4>
<p>
Для просмотра таблицы маршрутизации используйте команду <code>ip route</code>:
</p>
<pre><code>
ip_route_show.sh
</code></pre>
<p>
Пример вывода:
</p>
<pre><code>
route_table_example.txt
</code></pre>

<h4>Добавление маршрута</h4>
<p>
Для добавления маршрута используйте команду <code>ip route add</code>:
</p>
<pre><code>
ip_route_add.sh
</code></pre>

<h4>Удаление маршрута</h4>
<p>
Для удаления маршрута используйте команду <code>ip route del</code>:
</p>
<pre><code>
ip_route_del.sh
</code></pre>

<h4>Изменение маршрута</h4>
<p>
Для изменения маршрута сначала удалите старый маршрут, а затем добавьте новый:
</p>
<pre><code>
ip_route_change.sh
</code></pre>

---
<h3>Работа с несколькими таблицами маршрутизации</h3>

<h4>Просмотр всех таблиц маршрутизации</h4>
<p>
Для просмотра всех таблиц маршрутизации используйте команду:
</p>
<pre><code>
ip_route_show_all.sh
</code></pre>

<h4>Создание новой таблицы маршрутизации</h4>
<p>
Для создания новой таблицы маршрутизации отредактируйте файл <code>/etc/iproute2/rt_tables</code>:
</p>
<pre><code>
rt_tables_example.txt
</code></pre>

<h4>Добавление маршрута в пользовательскую таблицу</h4>
<p>
Для добавления маршрута в пользовательскую таблицу используйте команду:
</p>
<pre><code>
ip_route_add_table.sh
</code></pre>

<h4>Использование правил маршрутизации</h4>
<p>
Правила маршрутизации определяют, какую таблицу маршрутизации использовать для того или иного пакета.
Для просмотра правил используйте команду:
</p>
<pre><code>
ip_rule_show.sh
</code></pre>
<p>
Пример вывода:
</p>
<pre><code>
ip_rule_example.txt
</code></pre>

<h4>Добавление правила маршрутизации</h4>
<p>
Для добавления правила маршрутизации используйте команду:
</p>
<pre><code>
ip_rule_add.sh
</code></pre>

<h4>Удаление правила маршрутизации</h4>
<p>
Для удаления правила маршрутизации используйте команду:
</p>
<pre><code>
ip_rule_del.sh
</code></pre>

<h3>Практические примеры</h3>

<h4>Пример: Настройка статического маршрута</h4>
<p>
Предположим, у вас есть две сети: <code>192.168.1.0/24</code> и <code>10.0.0.0/8</code>, соединённые через роутер с IP-адресом <code>192.168.1.2</code>.
Чтобы настроить маршрут к сети <code>10.0.0.0/8</code>, выполните:
</p>
<pre><code>
static_route_example.sh
</code></pre>

<h4>Пример: Настройка маршрута по умолчанию</h4>
<p>
Для настройки маршрута по умолчанию через шлюз <code>192.168.1.1</code> выполните:
</p>
<pre><code>
default_route_example.sh
</code></pre>

<h4>Пример: Использование нескольких таблиц маршрутизации</h4>
<p>
Предположим, у вас есть два интернет-провайдера с разными шлюзами.
Вы хотите использовать первый провайдер для большинства трафика, а второй — для определённых адресов.
</p>
<ol>
  <li>Отредактируйте файл <code>/etc/iproute2/rt_tables</code>:</li>
</ol>
<pre><code>
rt_tables_multi_isp.txt
</code></pre>
<ol start="2">
  <li>Добавьте маршруты в таблицы:</li>
</ol>
<pre><code>
ip_route_multi_isp.sh
</code></pre>
<ol start="3">
  <li>Добавьте правила маршрутизации:</li>
</ol>
<pre><code>
ip_rule_multi_isp.sh
</code></pre>

</ul>

<h3>Особенности и подводные камни</h3>

<h4>Проблемы с маршрутизацией</h4>
<p>
Если маршрут к получателю не найден, пакет будет отброшен. Проверьте:
</p>
<ul>
  <li>Таблицу маршрутизации (<code>ip route</code>).</li>
  <li>Настройки шлюза.</li>
  <li>Правила маршрутизации (<code>ip rule</code>).</li>
</ul>

<h4>Проблемы с интерфейсами</h4>
<p>
Если сетевой интерфейс не работает, проверьте:
</p>
<ul>
  <li>Физическое подключение (для Ethernet).</li>
  <li>Настройки IP-адреса (<code>ip a</code>).</li>
  <li>Статус интерфейса (<code>ip link</code>).</li>
  <li>Драйверы устройств.</li>
</ul>
<pre><code>
check_interface.sh
</code></pre>

<h4>Конфликты маршрутов</h4>
<p>
Если в таблице маршрутизации есть конфликтующие маршруты, используйте метрики для управления приоритетами:
</p>
<pre><code>
metric_example.sh
</code></pre>

<h4>Проблемы с IPv6</h4>
<p>
Если IPv6 не работает, проверьте:
</p>
<ul>
  <li>Включён ли IPv6 в ядре:</li>
</ul>
<pre><code>
ipv6_check.sh
</code></pre>
<ul>
  <li>Настройки IPv6-адресов:</li>
</ul>
<pre><code>
ipv6_addr_check.sh
</code></pre>
