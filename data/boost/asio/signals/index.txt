<h2>Signals</h2>

<p><strong>Signals</strong> — это механизм Boost.Asio для обработки системных сигналов (SIGINT, SIGTERM, SIGHUP и др.) в асинхронной модели. Он позволяет корректно завершать сервер, освобождать ресурсы, останавливать соединения и выполнять graceful shutdown без блокирующих вызовов.</p>

<p>Boost.Asio предоставляет класс <strong>signal_set</strong>, который интегрирует системные сигналы в event loop. Это позволяет обрабатывать сигналы так же, как любые другие асинхронные события.</p>

<h3>Теоретические основы</h3>

<p>Основные возможности signal_set:</p>

<ul>
  <li>подписка на один или несколько сигналов;</li>
  <li>асинхронное ожидание через <strong>async_wait()</strong>;</li>
  <li>возможность повторной подписки после обработки;</li>
  <li>интеграция с io_context и event loop;</li>
  <li>использование для graceful shutdown.</li>
</ul>

<p>Типичные сигналы:</p>

<ul>
  <li><strong>SIGINT</strong> — Ctrl+C;</li>
  <li><strong>SIGTERM</strong> — стандартное завершение процесса;</li>
  <li><strong>SIGHUP</strong> — перезагрузка конфигурации (Linux/Unix).</li>
</ul>

<p>Типичный сценарий:</p>

<ol>
  <li>Создать <strong>signal_set</strong>;</li>
  <li>Добавить нужные сигналы;</li>
  <li>Вызвать <strong>async_wait()</strong>;</li>
  <li>При получении сигнала — выполнить shutdown логики;</li>
  <li>Остановить io_context.</li>
</ol>

<h3>Практические рекомендации</h3>

<ul>
  <li>Используйте signals для корректного завершения сервера.</li>
  <li>После обработки сигнала можно повторно вызвать async_wait для перехвата следующих сигналов.</li>
  <li>В многопоточных приложениях используйте strand для синхронизации shutdown-логики.</li>
  <li>При завершении сервера вызывайте <strong>stop_all()</strong> у Connection Manager.</li>
  <li>Не вызывайте std::signal() — используйте только signal_set.</li>
</ul>

<h3>Особенности и подводные камни</h3>

<ul>
  <li>На Windows поддерживаются только SIGINT и SIGTERM.</li>
  <li>Если не вызвать io_context.stop(), сервер продолжит работать.</li>
  <li>Если не остановить активные сессии — возможны утечки.</li>
  <li>signal_set должен жить дольше, чем async_wait.</li>
</ul>

<h3>Заключение</h3>

<p>Signals — важный компонент Boost.Asio, позволяющий корректно завершать сервер и управлять жизненным циклом приложения. Он интегрирует системные сигналы в асинхронную модель, обеспечивая безопасный и предсказуемый shutdown.</p>
