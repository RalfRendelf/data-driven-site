<h2>TCP Server</h2>

<p><strong>TCP Server</strong> — это компонент, который принимает входящие соединения, создаёт сессии для каждого клиента и управляет их жизненным циклом. В Boost.Asio TCP‑сервер строится на основе <strong>tcp::acceptor</strong> и использует асинхронные операции для высокой производительности и масштабируемости.</p>

<p>Сервер работает по простому принципу: принимает соединение → создаёт session → передаёт управление Connection Manager → продолжает принимать новые соединения. Такой подход позволяет обслуживать тысячи клиентов одновременно.</p>

<h3>Теоретические основы</h3>

<p>Основные элементы TCP‑сервера:</p>

<ul>
  <li><strong>tcp::acceptor</strong> — принимает входящие соединения;</li>
  <li><strong>session</strong> — объект, обрабатывающий одно соединение;</li>
  <li><strong>connection_manager</strong> — управляет всеми активными сессиями;</li>
  <li><strong>strand</strong> — обеспечивает потокобезопасность;</li>
  <li><strong>io_context</strong> — event loop сервера.</li>
</ul>

<p>Типичный TCP‑сервер выполняет следующие шаги:</p>

<ol>
  <li>Создаёт <strong>io_context</strong>.</li>
  <li>Создаёт <strong>tcp::acceptor</strong> и привязывает его к порту.</li>
  <li>Запускает асинхронный цикл <strong>async_accept()</strong>.</li>
  <li>Для каждого клиента создаёт <strong>session</strong>.</li>
  <li>Передаёт session в <strong>Connection Manager</strong>.</li>
  <li>Продолжает принимать новые соединения.</li>
</ol>

<h3>Практические рекомендации</h3>

<ul>
  <li>Используйте <strong>async_accept()</strong> в цикле — сервер должен работать непрерывно.</li>
  <li>Каждая session должна иметь свой <strong>strand</strong> в многопоточном сервере.</li>
  <li>Connection Manager должен управлять временем жизни всех сессий.</li>
  <li>Не блокируйте поток внутри обработчиков — это замедляет сервер.</li>
  <li>Используйте несколько потоков с <strong>io_context.run()</strong> для масштабирования.</li>
</ul>

<h3>Особенности и подводные камни</h3>

<ul>
  <li>Если забыть вызвать accept() повторно — сервер примет только одно соединение.</li>
  <li>Если не использовать strand — возможны гонки при чтении/записи.</li>
  <li>Если не удалять завершённые сессии — будет утечка памяти.</li>
  <li>При ошибке accept() сервер должен продолжать работу.</li>
</ul>

<h3>Заключение</h3>

<p>TCP‑сервер в Boost.Asio — мощный и гибкий инструмент для построения высокопроизводительных сетевых приложений. Он обеспечивает асинхронную обработку соединений, масштабируемость и предсказуемость поведения. Правильная архитектура сервера включает Session, Connection Manager и использование strand для потокобезопасности.</p>
