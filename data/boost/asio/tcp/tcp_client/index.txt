<h2>TCP Client</h2>

<p><strong>TCP Client</strong> — это компонент, который устанавливает соединение с TCP‑сервером, отправляет данные и получает ответы. В Boost.Asio TCP‑клиент строится на основе <strong>tcp::socket</strong> и использует асинхронные операции для неблокирующей работы.</p>

<p>Клиент может работать в синхронном или асинхронном режиме, но в производственных приложениях почти всегда используется асинхронный подход. Он позволяет выполнять несколько операций одновременно, не блокируя поток.</p>

<h3>Теоретические основы</h3>

<p>Основные элементы TCP‑клиента:</p>

<ul>
  <li><strong>tcp::resolver</strong> — преобразует доменное имя в IP‑адрес;</li>
  <li><strong>tcp::socket</strong> — основной объект для чтения/записи;</li>
  <li><strong>async_connect</strong> — асинхронное подключение к серверу;</li>
  <li><strong>async_read / async_write</strong> — асинхронный обмен данными;</li>
  <li><strong>strand</strong> — обеспечивает потокобезопасность;</li>
  <li><strong>reconnect‑логика</strong> — повторное подключение при ошибках.</li>
</ul>

<p>Типичный TCP‑клиент выполняет следующие шаги:</p>

<ol>
  <li>Создаёт <strong>io_context</strong>.</li>
  <li>Разрешает адрес сервера через <strong>resolver</strong>.</li>
  <li>Подключается к серверу.</li>
  <li>Отправляет запрос.</li>
  <li>Получает ответ.</li>
  <li>Обрабатывает ошибки и при необходимости переподключается.</li>
</ol>

<h3>Практические рекомендации</h3>

<ul>
  <li>Используйте <strong>resolver</strong> для подключения по доменному имени.</li>
  <li>В многопоточных клиентах используйте <strong>strand</strong> для защиты сокета.</li>
  <li>Обрабатывайте ошибки подключения — сервер может быть временно недоступен.</li>
  <li>Используйте <strong>async_read</strong> и <strong>async_write</strong> для неблокирующей работы.</li>
  <li>Реализуйте reconnect‑механику для устойчивости клиента.</li>
</ul>

<h3>Особенности и подводные камни</h3>

<ul>
  <li>Если сервер закрыл соединение — async_read вернёт ec::eof.</li>
  <li>Нельзя выполнять несколько async_write одновременно без strand.</li>
  <li>При reconnect важно закрывать старый сокет.</li>
  <li>Не используйте блокирующие операции в event loop.</li>
</ul>

<h3>Заключение</h3>

<p>TCP‑клиент в Boost.Asio — гибкий и мощный инструмент для построения сетевых приложений. Он позволяет реализовать быстрый, надёжный и неблокирующий обмен данными с сервером. Правильная архитектура клиента обеспечивает устойчивость, масштабируемость и предсказуемость поведения.</p>
