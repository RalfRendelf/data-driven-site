<h2>POSIX Descriptors</h2>

<p><strong>POSIX Descriptors</strong> — это механизм Boost.Asio, позволяющий работать с произвольными файловыми дескрипторами POSIX: pipe, FIFO, eventfd, файловые дескрипторы устройств, сокеты, псевдотерминалы и любые другие объекты, поддерживающие неблокирующее чтение/запись.</p>

<p>Boost.Asio предоставляет класс <strong>posix::stream_descriptor</strong>, который оборачивает существующий файловый дескриптор и интегрирует его в асинхронную модель Asio. Это позволяет использовать единый event loop для сетевых сокетов, таймеров и любых POSIX-ресурсов.</p>

<h3>Теоретические основы</h3>

<p>Основные возможности posix::stream_descriptor:</p>

<ul>
  <li>обёртка над существующим файловым дескриптором (int fd);</li>
  <li>поддержка <strong>async_read_some</strong> и <strong>async_write_some</strong>;</li>
  <li>интеграция с io_context и event loop;</li>
  <li>поддержка неблокирующего ввода/вывода;</li>
  <li>работа с pipe, eventfd, файловыми дескрипторами устройств.</li>
</ul>

<p>Типичные сценарии использования:</p>

<ul>
  <li>интеграция pipe между потоками или процессами;</li>
  <li>обработка сигналов через signalfd (Linux);</li>
  <li>интеграция eventfd для уведомлений;</li>
  <li>работа с псевдотерминалами (pty);</li>
  <li>интеграция драйверов и устройств в Asio.</li>
</ul>

<h3>Практические рекомендации</h3>

<ul>
  <li>Используйте posix::stream_descriptor для интеграции любых системных дескрипторов в event loop.</li>
  <li>Убедитесь, что дескриптор переведён в неблокирующий режим.</li>
  <li>Закрывайте дескриптор только через stream_descriptor — не вручную.</li>
  <li>Для межпоточного взаимодействия используйте eventfd или pipe.</li>
  <li>В многопоточных приложениях используйте strand для защиты логики.</li>
</ul>

<h3>Особенности и подводные камни</h3>

<ul>
  <li>Некоторые дескрипторы не поддерживают неблокирующий режим — Asio работать не будет.</li>
  <li>При закрытии fd вручную возможен use-after-free внутри Asio.</li>
  <li>eventfd доступен только в Linux — на других системах используйте pipe.</li>
  <li>При чтении из pipe возможен EOF — это нормальное поведение.</li>
</ul>

<h3>Заключение</h3>

<p>POSIX Descriptors — мощный инструмент Boost.Asio, позволяющий интегрировать любые системные ресурсы в асинхронную модель. Это делает Asio универсальным event loop, способным работать не только с сетью, но и с любыми POSIX-дескрипторами.</p>
