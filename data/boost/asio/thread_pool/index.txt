<h2>Thread Pool</h2>

<p><strong>Thread Pool</strong> — это компонент Boost.Asio, предоставляющий пул потоков для выполнения задач, не связанных с вводом/выводом. Он используется для распределения CPU-bound работы, параллельной обработки данных и разгрузки основного event loop.</p>

<p>В отличие от <strong>io_context</strong>, который обрабатывает сетевые события, <strong>thread_pool</strong> предназначен для выполнения произвольных функций, вычислений и фоновых задач. Он идеально подходит для серверов, где нужно обрабатывать данные после получения от клиента.</p>

<h3>Теоретические основы</h3>

<p>Основные возможности thread_pool:</p>

<ul>
  <li>создание пула потоков фиксированного размера;</li>
  <li>выполнение задач через <strong>boost::asio::post()</strong>;</li>
  <li>автоматическое распределение задач между потоками;</li>
  <li>безопасное завершение через <strong>join()</strong>;</li>
  <li>интеграция с coroutines и executors.</li>
</ul>

<p>Типичные сценарии использования:</p>

<ul>
  <li>обработка больших сообщений;</li>
  <li>парсинг JSON/XML;</li>
  <li>шифрование/дешифрование;</li>
  <li>CPU-bound вычисления;</li>
  <li>offload тяжёлой логики из session.</li>
</ul>

<p>Thread Pool не заменяет io_context — он дополняет его, позволяя разделить сетевую и вычислительную нагрузку.</p>

<h3>Практические рекомендации</h3>

<ul>
  <li>Используйте thread_pool для тяжёлых вычислений, чтобы не блокировать io_context.</li>
  <li>Размер пула выбирайте равным количеству ядер CPU.</li>
  <li>Не выполняйте сетевые операции внутри thread_pool — только CPU-bound.</li>
  <li>Для синхронизации используйте strand или lock-free структуры.</li>
  <li>Если задача тяжёлая — всегда offload в thread_pool.</li>
</ul>

<h3>Особенности и подводные камни</h3>

<ul>
  <li>Если выполнять CPU-bound задачи в io_context — сервер будет лагать.</li>
  <li>thread_pool не должен выполнять async-операции — это ответственность io_context.</li>
  <li>Не создавайте слишком большой пул — это приведёт к переключению контекста.</li>
  <li>Не забывайте вызывать <strong>pool.join()</strong> перед завершением программы.</li>
</ul>

<h3>Заключение</h3>

<p>Thread Pool — мощный инструмент Boost.Asio, позволяющий разгружать event loop и выполнять тяжёлые вычисления параллельно. Он идеально подходит для высоконагруженных серверов, где требуется разделение сетевой и вычислительной нагрузки.</p>
