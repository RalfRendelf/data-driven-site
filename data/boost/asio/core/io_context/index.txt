<h2>io_context</h2>

<p><strong>io_context</strong> — центральный объект Boost.Asio, управляющий выполнением всех асинхронных операций. Он содержит очередь задач, обработчиков, таймеров, сетевых событий и обеспечивает их выполнение в рамках event loop. Любое приложение на Boost.Asio начинается с создания io_context.</p>

<p>io_context выполняет три ключевые функции:</p>

<ul>
  <li>Хранит очередь задач и событий.</li>
  <li>Запускает event loop через <strong>run()</strong>.</li>
  <li>Распределяет обработчики между потоками.</li>
</ul>

<p>Все сетевые операции (async_read, async_write, async_accept, таймеры, SSL) привязаны к io_context и выполняются в его контексте.</p>

<h3>Теоретические основы</h3>

<p>Основные методы io_context:</p>

<ul>
  <li><strong>run()</strong> — запускает event loop и блокирует поток до завершения всех задач.</li>
  <li><strong>poll()</strong> — обрабатывает только готовые события и возвращает управление немедленно.</li>
  <li><strong>restart()</strong> — позволяет повторно использовать io_context после завершения run().</li>
  <li><strong>stop()</strong> — принудительно завершает event loop.</li>
  <li><strong>post()</strong> — помещает задачу в очередь на выполнение.</li>
  <li><strong>get_executor()</strong> — возвращает executor, управляющий выполнением задач.</li>
</ul>

<p>io_context может работать в одном или нескольких потоках. Если вызвать run() в нескольких потоках, они будут совместно обрабатывать очередь задач, что позволяет легко масштабировать сервер.</p>

<h3>Практические рекомендации</h3>

<ul>
  <li>Используйте <strong>несколько потоков</strong>, если приложение должно обрабатывать много соединений.</li>
  <li>Не вызывайте run() повторно без restart().</li>
  <li>Используйте <strong>post()</strong> для передачи задач между потоками.</li>
  <li>Если run() завершается сразу — проверьте, есть ли активные задачи или work_guard.</li>
  <li>io_context должен жить дольше всех объектов, использующих его executor.</li>
</ul>

<h3>Особенности и подводные камни</h3>

<ul>
  <li>Если нет активных задач — run() завершится немедленно.</li>
  <li>stop() завершает event loop, но не отменяет активные операции.</li>
  <li>Асинхронные операции могут завершаться синхронно — это нормально.</li>
  <li>При использовании нескольких потоков важно учитывать синхронизацию (strand).</li>
</ul>

<h3>Заключение</h3>

<p>io_context — фундамент Boost.Asio. Он управляет всеми асинхронными операциями, потоками и обработчиками. Глубокое понимание io_context необходимо для создания производительных и масштабируемых сетевых приложений.</p>
