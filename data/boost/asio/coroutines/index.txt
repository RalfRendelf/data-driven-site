<h2>Coroutines</h2>

<p>Boost.Asio поддерживает асинхронное программирование на основе корутин C++20. Корутины позволяют писать асинхронный код в линейном стиле, без вложенных callback-функций. Это делает код чище, проще и легче для сопровождения.</p>

<p>В Asio корутины работают через механизм <strong>awaitable&lt;T&gt;</strong> и оператор <strong>co_await</strong>. Любая асинхронная операция (async_read, async_write, async_accept) может быть выполнена в корутине, если используется соответствующий completion token — <strong>use_awaitable</strong>.</p>

<h3>Теоретические основы</h3>

<p>Основные элементы корутин в Boost.Asio:</p>

<ul>
  <li><strong>awaitable&lt;T&gt;</strong> — возвращаемый тип корутины.</li>
  <li><strong>co_await</strong> — оператор ожидания результата асинхронной операции.</li>
  <li><strong>use_awaitable</strong> — completion token, включающий поддержку корутин.</li>
  <li><strong>co_spawn</strong> — запуск корутины в io_context.</li>
  <li><strong>detached</strong> — политика завершения корутины без ожидания результата.</li>
</ul>

<p>Корутины позволяют писать асинхронный код так же, как синхронный:</p>

<pre><code>
auto n = co_await async_read(socket, buffer, use_awaitable);
</code></pre>

<p>При этом event loop продолжает работать, а поток не блокируется.</p>

<h3>Практические рекомендации</h3>

<ul>
  <li>Используйте корутины для сложных последовательностей async-операций.</li>
  <li>Не забывайте запускать корутины через <strong>co_spawn</strong>.</li>
  <li>Для многопоточных приложений используйте <strong>strand</strong> внутри корутин.</li>
  <li>Корутины не создают поток — они выполняются внутри event loop.</li>
  <li>Если корутина должна жить долго — используйте work_guard.</li>
</ul>

<h3>Особенности и подводные камни</h3>

<ul>
  <li>Исключения внутри корутин нужно обрабатывать вручную.</li>
  <li>Если io_context завершится — корутины тоже завершатся.</li>
  <li>Не используйте co_await внутри блокирующих операций.</li>
  <li>Корутины требуют C++20 и компилятора с поддержкой coroutine TS.</li>
</ul>

<h3>Заключение</h3>

<p>Корутины — современный и удобный способ работы с асинхронностью в Boost.Asio. Они позволяют писать компактный, читаемый и поддерживаемый код, сохраняя при этом высокую производительность и неблокирующую модель выполнения.</p>
