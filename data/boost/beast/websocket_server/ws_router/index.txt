<h2>WebSocket Router</h2>

<p><strong>WebSocket Router</strong> — это компонент WebSocket‑сервера, который отвечает за маршрутизацию входящих сообщений по типам, каналам или действиям. В отличие от HTTP Router, который работает с методами и путями, WebSocket Router обрабатывает сообщения в реальном времени и распределяет их между обработчиками в зависимости от структуры данных (обычно JSON) или бинарного протокола.</p>

<p>Router позволяет разделить логику WebSocket‑сервера на независимые обработчики: чат, уведомления, игровые события, команды управления, телеметрию и т. д. Это делает архитектуру масштабируемой, модульной и удобной для сопровождения.</p>

<h3>Теоретические основы</h3>

<p>WebSocket Router обычно работает поверх WebSocket Session и получает уже разобранное сообщение (строку или бинарный буфер). Основные задачи Router:</p>

<ul>
  <li>определить тип сообщения (например, по полю <code>type</code> в JSON);</li>
  <li>выбрать соответствующий обработчик;</li>
  <li>передать ему данные;</li>
  <li>вернуть ответ (если требуется);</li>
  <li>обрабатывать ошибки маршрутизации.</li>
</ul>

<p>В отличие от HTTP, WebSocket — это поток сообщений, поэтому Router должен быть быстрым, неблокирующим и потокобезопасным. В большинстве случаев используется JSON‑протокол, где каждое сообщение имеет структуру:</p>

<pre><code>
{
  "type": "chat",
  "payload": { ... }
}
</code></pre>

<p>Router может быть реализован как таблица обработчиков, map, словарь функций или цепочка middleware.</p>

<h3>Практические применения</h3>

<ul>
  <li>чат‑сервисы (типы: <code>chat</code>, <code>typing</code>, <code>join</code>);</li>
  <li>игровые события (<code>move</code>, <code>attack</code>, <code>sync</code>);</li>
  <li>уведомления (<code>alert</code>, <code>notify</code>);</li>
  <li>биржевые данные (<code>price</code>, <code>orderbook</code>);</li>
  <li>IoT‑телеметрия (<code>sensor</code>, <code>status</code>);</li>
  <li>управление устройствами (<code>command</code>, <code>control</code>).</li>
</ul>

<h3>Особенности и подводные камни</h3>

<ul>
  <li>Router должен быть быстрым — JSON‑парсинг может быть узким местом.</li>
  <li>Нельзя выполнять тяжёлые операции внутри обработчиков — используйте thread pool.</li>
  <li>Нужно валидировать входящие данные, иначе возможны ошибки и DoS.</li>
  <li>Middleware может сильно замедлить обработку — используйте осторожно.</li>
  <li>Необходимо логировать ошибки маршрутизации.</li>
  <li>При большом количестве типов сообщений лучше использовать enum + switch.</li>
</ul>

<h3>Заключение</h3>

<p>WebSocket Router — ключевой компонент WebSocket‑сервера, позволяющий структурировать логику обработки сообщений. Он обеспечивает модульность, расширяемость и удобство сопровождения. Использование Router особенно важно в real‑time системах, где множество типов сообщений должны обрабатываться быстро и надёжно.</p>
