<h2>CMake — современная система сборки C++ проектов</h2>

<p>CMake — это кроссплатформенная система автоматизации сборки, предназначенная для генерации проектов под различные IDE и build‑системы: Make, Ninja, Visual Studio, Xcode и другие. Она является стандартом де‑факто для C++ проектов, начиная от небольших утилит и заканчивая крупными промышленными системами. Современный CMake (3.10+) использует target‑based API, обеспечивает строгую декларативность и позволяет гибко управлять зависимостями, конфигурациями и компиляторами.</p>

<p>В этой статье мы подробно разберём архитектуру CMake, ключевые команды, макросы, переменные, структуру проектов, подключение зависимостей, тестирование, смену компилятора, генераторы, а также типичные ошибки и best practices.</p>

<h3>Теоретические основы</h3>

<p>CMake использует собственный DSL (Domain Specific Language), который описывает цели сборки (<strong>targets</strong>), зависимости, параметры компиляции и конфигурации. В отличие от Makefile, CMake не выполняет сборку напрямую — он генерирует файлы для конкретной системы сборки.</p>

<p>Основные концепции CMake:</p>

<ul>
  <li><strong>Проект (project)</strong> — корневая сущность, определяющая язык, версию и параметры.</li>
  <li><strong>Цель (target)</strong> — исполняемый файл, библиотека или интерфейсная сущность.</li>
  <li><strong>Генератор</strong> — система сборки (Make, Ninja, MSVC, Xcode).</li>
  <li><strong>Конфигурации</strong> — Debug, Release, RelWithDebInfo, MinSizeRel.</li>
  <li><strong>Переменные</strong> — CMAKE_CXX_STANDARD, CMAKE_BUILD_TYPE, PROJECT_SOURCE_DIR.</li>
  <li><strong>Команды target_*</strong> — современный API для управления зависимостями.</li>
</ul>

<p>Современный CMake строится вокруг <strong>target-based</strong> подхода, где все свойства (флаги, include‑директории, библиотеки) привязываются к конкретным целям, а не глобально.</p>

<h3>Практические применения</h3>

<ul>
  <li>Сборка кроссплатформенных C++ проектов.</li>
  <li>Генерация проектов для IDE (CLion, Visual Studio, QtCreator).</li>
  <li>Интеграция CI/CD (GitHub Actions, GitLab CI, Jenkins).</li>
  <li>Подключение внешних библиотек (Boost, OpenSSL, fmt, spdlog).</li>
  <li>Создание модульных проектов с несколькими библиотеками.</li>
  <li>Организация тестирования через CTest.</li>
</ul>

<h3>Особенности и подводные камни</h3>

<ul>
  <li>Не смешивайте старый и новый CMake (include_directories vs target_include_directories).</li>
  <li>Не используйте глобальные флаги — привязывайте всё к target.</li>
  <li>Не используйте file(GLOB) для исходников — это ломает инкрементальную сборку.</li>
  <li>Не устанавливайте CMAKE_BUILD_TYPE в много-конфигурационных генераторах (MSVC).</li>
  <li>Используйте FetchContent вместо ручной загрузки библиотек.</li>
  <li>Используйте INTERFACE targets для header-only библиотек.</li>
</ul>

<h3>Заключение</h3>

<p>CMake — мощная, гибкая и современная система сборки, которая стала стандартом для C++ проектов. Она позволяет создавать кроссплатформенные решения, управлять зависимостями, подключать внешние библиотеки, организовывать тестирование и CI/CD. Современный target‑based подход обеспечивает чистую архитектуру и предсказуемое поведение сборки.</p>
