# 📘 data-driven-site

Прототип SPA‑документационного сайта на HTML/CSS/JS (ES‑модули) с динамической генерацией контента из JSON‑структур.  
В дальнейшем проект станет частью распределённой системы **CodeDocNet**:  
фронтенд превратится в Web UI‑клиент, взаимодействующий с сервером по TCP/JSON, а контент будет поступать из базы данных через центральный маршрутизатор и генератор страниц.

---

## 🔹 Возможности

- Категории (`patterns`, `architectures`, `stl`, `microcontrollers` и др.), каждая со своим `index.json`.
- Поиск по ключевым полям сразу во всех категориях.
- Адаптивный интерфейс:
  - таблицы на десктопе,
  - карточки на мобильных устройствах.
- Гибкая структура колонок, поддержка текстовых файлов с HTML‑разметкой.
- Модульная архитектура:
  - `main.js` — точка входа,
  - `generator.js` — генерация страниц,
  - `navigation.js` — меню и маршрутизация.
- Добавление новых категорий происходит через:
  1. запись в `categories.js`,
  2. создание папки с `index.json`,
  3. добавление `.txt` файлов с контентом.
- Папка `data/` содержит промт‑шаблоны для генерации `.txt` файлов в нужной структуре.

---

## ⚙️ Запуск

Сайт использует **ES‑модули**, поэтому требуется запуск через локальный HTTP‑сервер.

Перейдите в корневую папку проекта и выполните один из вариантов:

### 🟦 Вариант 1 — Python
```bash
python3 -m http.server
или
python -m http.server ваш порт(по умолчанию 8000)
```

### 🟥 Вариант 2 — Node.js
```bash
npx server
```
### Вариант 3 — VS Code
Использовать расширение Live Server.

### Вариант 4 — запуск через батник
Запустить файл run_server.bat (требуется установленный Python).
